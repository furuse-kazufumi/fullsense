# Singularity Requirements: 次世代AIの認知拡張に関する要件定義(スケルトン)

> **本書のステータス**: スケルトン。本気で立項するときに肉付けする。
> **本書の目的**: 人間の日常的次元の制約を超えた表現空間で AI が思考し、人間単独では到達できない知見と技術を生み出すための、**組織レベルの要件定義**。
> **本書と `HYPERDIMENSIONAL_THINKING.md` の関係**: 後者は本書の思想を日々の開発実務に落とすための指針。本書が「何を目指すか」、後者が「日々どう振る舞うか」。

---

## 0. 立項の判断基準

本書を正式な要件定義として立ち上げるかどうかは、以下が揃ったときに判断する:

- [ ] 個別プロジェクト(llmesh, llive, raptor等)が `HYPERDIMENSIONAL_THINKING.md` の基本原則を実装段階で運用できている
- [ ] 「人間が日常次元で考えただけでは得られない結論」が、少なくとも一つの具体例として出ている
- [ ] 組織として、個別プロジェクトの成果物ではなく **AIの思考様式そのもの** を開発対象に据える合意がある
- [ ] 評価指標(非自明性テスト)を運用できる体制がある

上記が揃わないうちは、本書はスケルトンのまま保留する。

## 1. ビジョン

- 人間との関係性(代替・補完・共生のどれを目指すか)
- 到達点の判定基準

## 2. 背景と動機

[要記述] なぜ今これを立項する必要があるか。

検討事項:
- 既存のAI開発(LLM, VLM, 3DGS等)が暗黙に持つ次元的制約の具体例
- その制約が技術進歩のボトルネックになっている証拠
- 他の研究機関・企業の動向との差別化

---

## 3. 要件

### 3.1 機能要件

[要記述] AIが満たすべき認知能力。

検討事項:
- 表現空間の自律的選択能力
- 複数表現の同時保持と相互変換
- 不変量の自動発見
- 人間への翻訳層の自動構築
- 自己の認知過程のメタ認知

### 3.2 非機能要件

[要記述] 性能・運用・倫理。

検討事項:
- 計算資源(ローカル推論との両立)
- 説明可能性(人間理解可能な層への射影の品質)
- 安全性(認知拡張がもたらすリスクの管理)
- 検証可能性(出力された知見が本物か偽物かをどう判定するか)

### 3.3 評価要件

[要記述] 拡張思考が実現できているかを測る方法。

検討事項:
- 多重表現テストの体系化
- 転移テストの設計
- 還元テストの基準
- **非自明性テストの形式化**(最重要・最難)

---

## 4. アーキテクチャ方針

[要記述] 上記要件を満たすシステムの全体像。

検討事項:
- 表現空間ライブラリの設計(`HYPERDIMENSIONAL_THINKING.md` 3節を発展)
- AIエージェントへの空間選択権の委譲方式
- 複数AI間での認知の共有方式
- 人間とのインターフェイス層

## 6. リスクと制約

[要記述]

検討事項:
- 認知拡張がもたらす不可逆な変化のリスク
- 人間の理解を超えた知見が出たときの取り扱い
- 偽物の「拡張」(=人間にとって不可解だが本質的に無意味な出力)の検出
- 倫理的・社会的影響
- 安全性研究との接続
