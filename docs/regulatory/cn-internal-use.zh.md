## 1. 适用法规

本文档梳理企业内部 (非面向公众) 部署 FullSense (llmesh / llive / llove)
时, 下列中国法规如何适用, 以及为何「企业内部使用」模式通常可豁免主要
备案义务.

| 法规 | 施行日 | 关联度 |

| 《生成式人工智能服务管理暂行办法》 | 2023-08-15 | 主要 |

## 2. 「面向公众」与「内部使用」的边界

### 2.1 备案对象 — 面向公众的服务

《生成式人工智能服务管理暂行办法》第二条规定, 适用对象为「在境内向公众
提供生成式人工智能服务」的主体. 具有「**舆论属性或社会动员能力**」的服务
需要算法备案, 典型形态包括:

- 论坛 / 博客 / 微博 / 聊天室 / 通讯群组
- 公众账号 / 短视频 / 直播
- 信息分享 / 小程序
- 其他向公众提供意见表达通道或具备公众活动动员能力的互联网服务

### 2.2 豁免对象 — 内部使用

第二条第三款明确豁免下列情形:

> 「**业内单位、企业、教育和科研机构、公共文化机构、有关专业机构等研发、
> 应用生成式人工智能技术, 未向境内公众提供生成式人工智能服务的, 不适用
> 本办法的规定**.」

即: **企业仅供员工内部使用、生成内容仅在企业内部使用** 时, 本办法不适用
(算法备案、安全评估等义务也不触发).

### 2.3 边界判定指引

典型的「内部使用」(可豁免) 场景:

- AI 仅由员工访问, 生成内容仅在企业内部 (维基 / 代码 / 设计文档 / 业务
  备忘) 流转
- 内部研发 / 试验 / 原型设计
- 跨部门 / 跨厂区的内部工作流自动化
- 注: 若日后转为公众服务, 转换时刻产生备案义务

「面向公众」(需要备案) 场景:

- 面向一般消费者的 chatbot / 问答引擎
- 公众可访问的 web AI 功能
- 外部合作伙伴 / 客户企业员工可访问的 AI 服务
- 对外开放的 API 服务
- 称为「内部」但实际允许广泛公众访问

灰色地带 (各企业自行确认):

- B2B SaaS, 仅客户企业员工使用
- 子公司 / 关联公司 / 业务合作伙伴员工的访问

## 3. FullSense 部署模式与备案适用性

### 3.1 典型 (内部使用、豁免)

- **llmesh** 部署在企业内 (on-prem) 作为内部 LLM hub, 服务于员工工作效率.
- **llive** 用于生成内部文档 / 代码 / Brief, 输出仅入内部系统.
- **llove** 作为开发者的可视化 / 观测 IDE.

只要访问控制与上述模式一致, 即可对应「内部使用」豁免.

### 3.2 转为公众服务 (转换时刻备案)

若上述功能后续对外开放 (公众网 / 合作客户 / 一般消费者), 转换时即产生
备案义务. 转换计划应与法务一并规划.

### 3.3 FullSense 架构层面对应

| 监管要求 (内部使用亦应具备) | FullSense 功能 | 状态 |
|---|---|---|
| 审计日志 (修正网络安全法 2026-01) | llmesh + llive 审计日志 + Approval Bus | 部分实装 |
| 人工监督 | llive HITL Approval Bus | 已实装 |
| 输出来源追踪 | llive OKA-FX + Annotation Channel | 部分实装 |
| 数据出境管控 | on-prem 完结架构 | 已实装 |
| 公众服务用安全评估报告 | (仅当对外开放时需要) | 未实装 (低优先) |

## 4. 推荐运维 (维持内部使用豁免)

### 4.1 访问控制

- AI 服务部署于**企业内网 / VPN** 之内.
- 通过内部 SSO / LDAP / Active Directory 进行认证.
- 若需向合作伙伴开放访问, 通过明确合同 + 访问日志加以约束.
- 不在已备案前对外开放 API.

### 4.2 审计日志

- 全部 AI 请求 / 响应保留 **6 个月以上** (修正网络安全法); 若同时受
  EU AI Act 约束, 保留 6 年.
- 将 llmesh 审计日志 + llive Approval Bus 日志接入生产环路.
- 法务 / 审计部门定期抽样审查.

### 4.3 输出内部化

- 制定政策:「**生成内容未经人工审核不对外公开**」.
- HITL Approval Bus 作为「生成」到「发布」之间的关卡.
- 内部 wiki / 文档 / 代码摄入可直接进行; 推送至公众网需要审核.

### 4.4 监管跟踪

- 法务部门定期跟踪修正网络安全法 (2026-01) AI 条款以及《人工智能拟人化
  互动服务管理暂行办法》(2026-07-15 拟施行) 的进展.
- 监管文本变化时, 同步更新企业运维政策.

## 5. 转为公众服务时 (供参考)

若计划进入公众服务阶段, 需准备:

- 通过 CAC 算法备案系统在 **10 个工作日内** 完成算法备案.
- 「舆论属性或社会动员能力」服务需要安全评估.
- 大模型备案 (如适用).
- 用户协议 / 投诉举报机制 / 内容审核 / 用户实名认证.
- 深度合成内容标识 (针对 AI 生成图像 / 音频 / 视频).

截至 2025-12-31, 国家网信办公开数据显示 **748 项** 生成式 AI 服务、
**435 项** 应用 / 功能已完成备案.

## 6. 各企业自查清单

- [ ] 确认本企业 AI 使用「未向境内公众提供」.
- [ ] 确认子公司 / 关联公司 / 合作伙伴的访问不构成「公众」.
- [ ] 遵守修正网络安全法 AI 条款 (审计日志等).
- [ ] 准备 2026-07-15 拟施行的拟人化互动 AI 弁法 (若部署对话型 AI).
- [ ] 行业层面附加监管 (金融 / 医疗 / 教育 / 政府等).
- [ ] 数据出境管控 (个人信息 / 重要数据的境外移转禁止).
- [ ] 行业协会 / 监管机构的额外指引.
