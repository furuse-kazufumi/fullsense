# 古典原理 → 差別化 trait マッピング — 孫子の兵法・論語

> **背景 (2026-06-06)**: 計算リソース制約下で大手 (OpenAI / Anthropic / Google)
> に正面から追いつけない前提のもと、差別化 trait を**できるだけたくさん積層**し
> 「特別な DNA を持つ AI」(quality-diversity 的ニッチ種) を目指す戦略の一環。
>
> **本ドキュメントの位置づけ**: raptor には既に
> [`tiers/personas/work_discipline.md`] と [`tiers/methodology/{sun_tzu_art_of_war,confucius_analects}.md`]
> があるが、それらは**「個々の作業の進め方の規律 (タスク discipline)」**に古典を写像
> したもの。本ドキュメントはその**先**を行き、古典原理を **FullSense/llcore の
> プロダクト・研究戦略・システム設計遺伝子**に翻訳する (アーキテクチャ level の DNA)。
> 重複ではなく、同じ原典を別の翻訳先 (戦略・設計遺伝子) に展開する。
>
> 直前に進化派 3 名のペルソナを llive `PERSONA_ONTOLOGY` に追加した (commit bbc4892)
> のと同じパターンで、古典思想家 (`sun-tzu-strategist` / `confucius`) も ontology に
> 追加済み (本セッション)。本ドキュメントはその思想的接地を記す正本。

## なぜ「計算力で複製できない trait」なのか — 戦略の核

大手の優位は **scale (パラメータ・データ・FLOPs)** に集中する。scale で買えるのは
「平均的に賢い」汎用性であって、以下は買えない:

1. **文化・規律としての trait** — 「証明してから採用する」「異常な結果を疑う」
   「確証段階を明示する」は、モデルの賢さでなく**運用文化**に宿る。GPU を 10 倍に
   しても自動では生えない。
2. **構造に焼き込んだ trait** — Approval Bus / 形式証明ゲート / SPC 管理図のように
   **アーキテクチャに不可逆性を持たせた**設計判断は、推論能力の高さとは独立。
3. **ニッチ選択そのもの** — 「大手の手薄な軸を突く」(避実撃虚) は scale を**避ける**
   戦略なので、定義上 scale で複製されにくい。

孫子と論語は、この 3 種の trait の**思想的源泉**として極めて適合する。孫子は
「資源で劣る側がいかに勝つか」を体系化した戦略書であり、まさに我々の状況に対応する。
論語は「知の誠実さ・自己修養・統治」を体系化しており、honest disclosure / 多層検問 /
記憶×推論の均衡に対応する。

## 凡例

各原理について以下を記す:
- **原文 (漢文) + 書き下し + かみくだき**
- **出典篇名** (孫子 13 篇 / 論語 篇名)
- **翻訳先**: (a) 研究戦略 / (b) システム設計遺伝子 / (c) 検問・統治 のいずれか
- **差別化への寄与**: なぜ大手がこの trait を計算力で複製できないか (1 行)

> 引用の正確性: 有名句のみを使用。出典曖昧なものは使わない。章句の篇名は
> web (ctext.org / 百度百科 / 漢文大系等) で裏取り済み。**孫子の篇名は十三篇の標準
> 区分** (計・作戦・謀攻・軍形・兵勢・虚実・軍争・九変・行軍・地形・九地・火攻・用間)。

# 第 I 部 — 孫子の兵法 (13 篇)

## S1. 廟算 — 算多きは勝つ

- **原文**: 「夫れ未だ戦わずして廟算して勝つ者は、算を得ること多ければなり。
  未だ戦わずして廟算して勝たざる者は、算を得ること少なければなり。」
- **書き下し / かみくだき**: 戦う前に廟堂で勝算を数え、勝算が多ければ勝ち、少なければ
  負ける。**事前計算 (廟算) の量が勝敗を決める**。
- **出典**: 計篇 (始計篇, 第 1)
- **翻訳先 (b) システム設計遺伝子**: llcore の**形式証明ゲート / null harness / 事前登録
  (pre-registration)** は廟算の現代版。変異を採用する前に「勝てる態勢 = 検証が通った
  状態」を数値で数える。PoC → フィジビリティ → 詳細設計の段階フローも廟算の精神。
- **差別化への寄与**: 「動く前に数える」は文化であり、推論能力では代替できない。
  大手は scale で「動きながら賢くなる」が、我々は動く前に証明する。

## S2. 先勝 — 勝兵は先ず勝ちて而る後に戦う

- **原文**: 「勝兵は先ず勝ちて而る後に戦いを求め、敗兵は先ず戦いて而る後に勝を求む。」
- **書き下し / かみくだき**: 勝つ軍はまず**勝てる態勢を作ってから**戦い、負ける軍は
  まず戦ってから勝ち筋を探す。
- **出典**: 軍形篇 (第 4)
- **翻訳先 (a) 研究戦略 + (b) 設計遺伝子**: **fail-closed gate の思想的接地**。
  「証明してから採用する」= 先勝。Z3 証明ゲート / 検証器が通ってから genome を populace
  に入れる。honest disclosure 文化 (異常に良い結果は内訳を疑う) も「戦ってから勝ちを
  探す」誘惑への防壁。
- **差別化への寄与**: 大手は「まず巨大モデルを走らせて後から評価」する余裕があるが、
  資源制約側の我々は**先に勝ち筋 (検証) を固定**することで無駄打ちを避ける = 制約が
  規律を生む。

## S3. 上兵は謀を伐つ — 戦わずして勝つ

- **原文**: 「上兵は謀を伐つ。其の次は交を伐つ。其の次は兵を伐つ。其の下は城を攻む。」
  「百戦百勝は善の善なる者に非ず。戦わずして人の兵を屈するは善の善なる者なり。」
- **書き下し / かみくだき**: 最上の戦は**相手の謀略を未然に挫く**こと。次が同盟を断つ、
  次が野戦、最下が攻城。戦わずに屈服させるのが最善。
- **出典**: 謀攻篇 (第 3)
- **翻訳先 (a) 研究戦略**: **正面決戦 (ベンチマーク総合スコアで大手に勝つ) を避け、
  問題そのものを作らない / 構造で無効化する**。Mythos 超え設計の核心
  (決定論オラクルが verifier-Goodhart を**構造的に回避** = そもそも騙されない) は
  「戦わずして勝つ」。攻城 (力押し) でなく謀略 (構造) で勝つ。
- **差別化への寄与**: scale 競争 (攻城戦) は資源で負ける。構造で問題を消す
  (謀を伐つ) は scale と独立な勝ち方。

## S4. 彼を知り己を知る

- **原文**: 「彼を知り己を知れば、百戦して殆うからず。彼を知らずして己を知れば、
  一勝一負す。彼を知らず己を知らざれば、戦う毎に必ず殆うし。」
- **書き下し / かみくだき**: 相手と自分の両方を知れば百回戦っても危うくない。片方だけ
  なら五分。両方知らねば必ず危うい。
- **出典**: 謀攻篇 (第 3)
- **翻訳先 (b) 設計遺伝子 + (c) 統治**: **二重把握の遺伝子**。「彼 = 対象 (コード・
  ベンチ・競合)」「己 = 自分の確証状態 (検証したか / 未確認か)」。llcore の社会的推論
  (相手モデルの心的状態推定) と honest disclosure (自分の確信度の明示) の両輪。
- **差別化への寄与**: 「己 (自分の不確かさ) を知る」メタ認知は、賢さでなく**自己監査
  文化**。大手の大モデルは過信しがち (己を知らない)。

## S5. 避実撃虚 — 実を避け虚を撃つ

- **原文**: 「兵の形は実を避けて虚を撃つ。」「水の地に因りて流れを制するが如く、
  兵は敵に因りて勝を制す。」
- **書き下し / かみくだき**: 軍の形は**敵の充実した所を避け、手薄な所 (虚) を撃つ**。
  水が地形に従って流れるように、兵は敵に応じて勝ち方を変える。
- **出典**: 虚実篇 (第 6)
- **翻訳先 (a) 研究戦略**: **ニッチ戦略の正本**。大手が厚い軸 (汎用ベンチ総合・大規模
  事前学習) を避け、手薄な軸を突く: ① on-prem $0 (cloud 課金の虚)、② 検証可能タスクで
  弱 on-prem を押し上げる (汎用推論の虚)、③ 主観・文化・表現領域 (検証不能ゆえ大手が
  避ける虚)、④ 形式証明 × 進化 (cross-family) という誰も埋めていない交点。
- **差別化への寄与**: 避実撃虚は**定義上 scale を避ける**戦略。大手が虚にしている軸を
  選ぶ限り、scale 増強では複製されない。

## S6. 兵は詭道なり — だが探索に限る

- **原文**: 「兵とは詭道なり。故に能なるも之に不能を示し、用なるも之に不用を示す。」
- **書き下し / かみくだき**: 戦とは**詭道 (敵を欺き惑わす術)**。できても出来ぬふりを
  し、使うのに使わぬふりをする。常識の逆を運用する。
- **出典**: 計篇 (始計篇, 第 1)
- **翻訳先 (b) 設計遺伝子**: **探索 (variation) は詭道で自由 — 常識外の変異 /
  novelty search / antifragile mutation で意外な手を試す**。ただし詭道は「探索」に限定し、
  「採用 (selection)」は後述の論語「仁」(C4/C5) で律する = **二層倫理**。進化の不確実性
  運用 = 詭道、採用ゲートの誠実 = 仁。
- **差別化への寄与**: 「探索は奇手・採用は誠実」という二層構造は、設計判断であって
  パラメータ数ではない。大手の安全層は単層 (出力フィルタ) になりがち。

## S7. 正合奇勝 — 正を以て合し、奇を以て勝つ

- **原文**: 「凡そ戦いは、正を以て合し、奇を以て勝つ。」
- **書き下し / かみくだき**: 戦は**正攻法 (正) で正面を固め、奇手 (奇) で勝負を決める**。
  正と奇は循環して尽きない。
- **出典**: 兵勢篇 (第 5)
- **翻訳先 (b) 設計遺伝子**: **「正 = 検証された安定コア」+「奇 = 進化派生集団の探索」**
  の二本立て。llcore の verified core (正) が地盤を固め、llive の派生集団進化 / 進化派
  ペルソナ (奇) が勝ち筋を探す。expert council の moderator (正) と participants (奇) も
  同型。正だけでは凡庸、奇だけでは不安定 — 循環させる。
- **差別化への寄与**: 「検証コア × 開放端探索」の同時保持は設計の妙であり、単に大きな
  モデルを回すこととは別物。

## S8. 勢 — 個の質でなく勢いで勝つ

- **原文**: 「善く戦う者は、之を勢に求めて人に責めず。故に能く人を択びて勢に任ぜしむ。」
- **書き下し / かみくだき**: 戦上手は**勝ちを「勢い (システム)」に求め、個々の兵の
  能力に責を負わせない**。適材を勢いに乗せる。
- **出典**: 兵勢篇 (第 5)
- **翻訳先 (b) 設計遺伝子**: **個々の小モデルの賢さでなく、集団 + 検証 + 文化の「勢」で
  勝つ**。弱い on-prem モデル (qwen2.5:14b 等) を、進化集団 + 決定論オラクル +
  multi-turn self-check という「勢」に乗せて押し上げる。FullSense ファミリー統合
  (llive↔llmesh↔llove) も個でなく勢。
- **差別化への寄与**: 「弱い個を強い系に乗せる」は scale の逆発想。大手は強い個 (大
  モデル) に頼るので、この勢の設計を持たない。

## S9. 兵に常勢なし — 形は水に象る

- **原文**: 「兵に常勢なく、水に常形なし。能く敵に因りて変化して勝を取る者、之を神と謂う。」
- **書き下し / かみくだき**: 軍に固定の態勢はなく、水に固定の形がない。**敵に応じて
  変化して勝つ者を神 (達人) という**。
- **出典**: 虚実篇 (第 6)
- **翻訳先 (b) 設計遺伝子**: **固定アーキテクチャを持たず、地形 (タスク・環境) に応じて
  kernel / persona / 戦略を切り替える**。kernel 多様化 (rwkv/mamba/hopfield/linear_attn) /
  meta-cognition (UCB1 で戦略選択) / persona composition mutation は「水の如き可塑性」。
- **差別化への寄与**: 固定大モデルでなく可塑的な系であること自体が trait。scale 増強は
  固定モデルを大きくするだけで、可塑性は与えない。

## S10. 拙速 — 速さは工程削減で得る

- **原文**: 「兵は拙速を聞くも、未だ巧遅を睹ざるなり。」
- **書き下し / かみくだき**: 戦は**多少拙くても速い**のは聞くが、巧みでも遅いのが
  良かった例はない。
- **出典**: 作戦篇 (第 2)
- **翻訳先 (b) 設計遺伝子**: **予測符号化アーキテクチャ (warning-zone 先回り生成 + 予測
  誤差 typed diff だけ push) で負レイテンシ**。速さは「賢くなる」でなく「無駄な工程
  (フル再生成・冗長ペイロード) を省く」ことで得る。ただし**拙速 ≠ 検証省略** (S2 と
  表裏: 速くても先勝の検証は省かない)。
- **差別化への寄与**: 負レイテンシ表現は scale でなくアーキ設計から来る。大モデルは
  むしろ遅くなる方向。

## S11. 火攻・慎重 — 利に非ざれば動かず

- **原文**: 「利に非ざれば動かず、得るに非ざれば用いず、危うきに非ざれば戦わず。」
  「主は怒りを以て師を興すべからず、将は慍りを以て戦いを致すべからず。」
- **書き下し / かみくだき**: **利がなければ動かず**、得るものがなければ兵を用いず、
  危機でなければ戦わない。怒りで兵を起こすな。
- **出典**: 火攻篇 (第 12)
- **翻訳先 (c) 統治**: **不可逆操作 (rm / push / DB drop) と GPU 投資判断は「利・得・危」
  が揃わねば動かさない**。GPU portfolio decision を「ユーザー判断」に留保したのは
  まさにこれ (感情・勢いで大金を投じない)。Approval Bus が「利に非ざれば動かず」を
  構造化する。
- **差別化への寄与**: 「動かない判断」を構造に持つこと。大手は資源があるので「とりあえず
  動かす」が許される — 我々の慎重は制約由来の規律。

## S12. 用間 — 先知は鬼神に求めず

- **原文**: 「先知なる者は、鬼神に取るべからず、事に象るべからず、度に験すべからず。
  必ず人に取りて、敵の情を知る者なり。」
- **書き下し / かみくだき**: **先んじて知る (先知) は、神頼みでも経験則でもなく、必ず
  人 (間者) から敵情を得る**。情報は一次ソースから取る。
- **出典**: 用間篇 (第 13)
- **翻訳先 (a) 研究戦略 + (c) 統治**: **corpus 先行 / 一次情報主義**。外部 AI (Codex/
  Gemini/Copilot) の finding は実コード・一次情報で一件ずつ検証 (鵜呑み禁止)。RAD 21
  分野コーパス / hacker_corpus は「間者の収集した敵情」。link-rot を Lychee CI で監視
  するのも諜報網の保全。
- **差別化への寄与**: 「先に情報を仕込む」corpus-first 優位は、推論ではなく**蓄積**。
  大手も持つが、ニッチ分野では小回りの効く我々が先行できる (corpus_first_advantage)。

# 第 II 部 — 論語 (孔子)

## C1. 温故知新 — 故きを温ねて新しきを知る

- **原文**: 「故きを温ねて新しきを知れば、以て師と為るべし。」(温故而知新、可以為師矣)
- **書き下し / かみくだき**: **古いものを温め直して新しい知見を得られれば、人の師に
  なれる**。過去を踏まえて新を生む。
- **出典**: 為政篇
- **翻訳先 (a) 研究戦略**: **corpus 先行 + 来歴 (provenance) 重視**。新機能の着手前に
  rad-research で先行研究を確認、git log / 過去 memory / 教訓を温ねてから新手へ。
  llcore の「過去の negative を新しい高次元条件で統一説明」も温故知新。
- **差別化への寄与**: 「過去を構造的に蓄積し再利用する」文化。大手はモデルを作り直す
  傾向 (温故より知新偏重) — 我々は来歴を資産化する。

## C2. 過則勿憚改 — 過ちて改むるに憚ること勿れ

- **原文**: 「過ちては則ち改むるに憚ること勿かれ。」(過則勿憚改)
  併せて「過ちて改めざる、是を過ちと謂う。」
- **書き下し / かみくだき**: **間違えたら改めるのをためらうな**。間違いを改めないことを
  こそ本当の過ちという。
- **出典**: 学而篇 (過則勿憚改) / 衛霊公篇 (過而不改是謂過矣)
- **翻訳先 (b) 設計遺伝子**: **honest disclosure の思想的接地**。異常に良い結果・stale な
  記録・over-claim に気づいたら即訂正 (L3 §3c の「84%→107%」訂正、B1「実は修正済み」
  訂正)。失敗実験を削除せず教訓として残す。閉ループ修正の文化。
- **差別化への寄与**: 「自分の誤りを開示する」のは賢さでなく規律。RLHF で「自信ありげに
  答える」よう最適化された大モデルとは逆方向の trait。

## C3. 学思並重 — 学びて思わざれば則ち罔し

- **原文**: 「学びて思わざれば則ち罔し、思いて学ばざれば則ち殆し。」
- **書き下し / かみくだき**: **学んで考えなければ身につかず (罔)、考えて学ばなければ
  独断で危うい (殆)**。記憶と推論は両輪。
- **出典**: 為政篇
- **翻訳先 (b) 設計遺伝子**: **記憶 × 推論の均衡 = 4 層メモリ × 10 思考因子**。学 (4 層
  メモリへの蓄積・retrieval) と思 (思考因子による推論・再構成) を両輪で回す。
  どちらかに偏った系 (RAG だけ / 推論だけ) は罔または殆。
- **差別化への寄与**: 「記憶アーキテクチャと推論プロセスを構造的に結合」する設計。
  大手はパラメータ内記憶に寄る (思 > 学) ので、明示的 4 層メモリ × 因子の均衡は独自。

## C4. 君子は和して同ぜず

- **原文**: 「君子は和して同ぜず、小人は同じて和せず。」
- **書き下し / かみくだき**: **君子は調和するが付和雷同しない**。小人は同調するが
  真に和さない。
- **出典**: 子路篇
- **翻訳先 (c) 検問・統治**: **多視点 review / Approval Bus / expert council**。
  10 思考因子の多視点を保ちつつ安易に同調しない。consensus 機構 (multi-agent) は
  「和」だが、collusion 検出 (coevolution_governance) で「同 (談合)」を防ぐ。
  「三人行えば必ず我が師有り、其の善き者を択ぶ」(述而篇) = 外部 AI レビューを謙虚に
  取り入れつつ善悪を選ぶ。
- **差別化への寄与**: 「合意しつつ談合しない」ガバナンスは構造設計。単一大モデルには
  多視点の内在的緊張がなく、和して同ずる (echo chamber) に陥りやすい。

## C5. 知之為知之 — 知るを知るとし、不知を不知とす

- **原文**: 「之を知るを之を知ると為し、知らざるを知らずと為す、是れ知るなり。」
- **書き下し / かみくだき**: **知っていることを知っているとし、知らないことを知らない
  とする。それが「知る」ということ**。
- **出典**: 為政篇
- **翻訳先 (a) 研究戦略 + (c) 統治**: **確証段階の明示 (suspicion → corroboration →
  reproduced → ...)**。未検証を「未確認」と明示する。llcore の evidence ladder /
  「実装済/未配線/部分実装/未実装」4-tier status は「知/不知」の構造化。captone の
  over-claim 批評で「N/A (構造的)」「still_inconclusive」を残すのもこれ。
- **差別化への寄与**: 「分からないことを分からないと言える」較正 (calibration) は、
  hallucination の対極。RLHF で過信方向に寄った大モデルが最も苦手とする trait。

## C6. 工欲善其事必先利其器 — 道具を研ぐ

- **原文**: 「工、其の事を善くせんと欲すれば、必ず先づ其の器を利くす。」
- **書き下し / かみくだき**: **良い仕事をしたければ、まず道具を研げ**。
- **出典**: 衛霊公篇
- **翻訳先 (b) 設計遺伝子**: **着手前に環境・harness・再現手順を整える**。RAPTOR の
  run-lifecycle / coverage tracking / SWD (strict write discipline) / plugin-integrity は
  「器を研ぐ」インフラ。pre-registration の harness validity 検証も同精神。鈍った道具
  (壊れた launcher) のまま突進しない。
- **差別化への寄与**: 検証可能性・監査可能性を**先に**構築する文化。大手は能力で押すが、
  我々は道具 (検証インフラ) の鋭さで差をつける。

## C7. 名正しからざれば言順わず — 正名

- **原文**: 「名正しからざれば則ち言順わず、言順わざれば則ち事成らず。」
- **書き下し / かみくだき**: **名 (定義・呼称) が正しくなければ言葉が筋立たず、言葉が
  筋立たねば事が成らない**。
- **出典**: 子路篇
- **翻訳先 (b) 設計遺伝子 + (c) 統治**: **status 値・命名・仕様の正確さ**。
  RAPTOR の OUTPUT STYLE (status は snake_case / Title Case を統一、ALL_CAPS 禁止)、
  ll- ファミリー名の衝突監査 (llmesh vs HPE)、RepIR→llrepr 改名は「正名」。曖昧な名の
  まま議論を積まない。
- **差別化への寄与**: 命名・仕様の規律は監査可能性の土台。MCP 信頼境界での再検証
  (fail-closed) も「名 = スキーマを正す」こと。scale とは無縁の設計品質。

## C8. 欲速則不達 — 速やかならんと欲すれば則ち達せず

- **原文**: 「速やかならんと欲すれば則ち達せず、小利を見れば則ち大事成らず。」
- **書き下し / かみくだき**: **急ぎすぎると達せず、目先の小利を追うと大事を成せない**。
- **出典**: 子路篇
- **翻訳先 (a) 研究戦略**: **検証を飛ばす「速さ」は達しない (孫子 S10 拙速と表裏)**。
  ベンチで小利 (見かけのスコア) を追って honest disclosure を破ると大事 (信頼) を失う。
  feedback_benchmark_honest_disclosure / measurement_purity (on-prem 純度を cloud と
  混ぜない) はこの戒め。
- **差別化への寄与**: 「測定の純度を守る」自制。スコア競争で大手と張り合わず、
  正しく測ることに価値を置く文化は scale で買えない。

**実装への接続提案**:
- G14/G15 は llive の `coevolution_governance` + Approval Bus + llcore 証明ゲートを
  **一つの「先勝 + 二層倫理」ポリシーオブジェクト**として束ねると trait として可視化
  できる (進化ゲノムの fitness にも「先勝度 = 採用前検証カバレッジ」を加点項目化)。
- G16 は QD の niche fitness に「大手盲点スコア」(検証可能性 × on-prem 純度 ×
  cross-family 脱相関) を behavioral descriptor として追加する形で実装可能。
- G17 は既存の expert council / 集団進化を「勢」メトリクス (個の弱さ × 系の強さ =
  弱 baseline からの押し上げ幅) で評価する観点を導入。
- G18 は既存 SWD/plugin-integrity を「正名 + 利器」遺伝子座として doc/監査で明示するだけ
  でよい (機構は既存)。

**追加した ontology ペルソナとの対応**:
- `sun-tzu-strategist` (孫子戦略家) = G14 (先勝) / G16 (避実撃虚) / G17 (勢) の人格化。
  factor_affinity: 不確実性・整合・閉ループ・現実接続が高い。
- `confucius` (孔子) = G14 (知/不知) / G15 (採用の仁) / G18 (正名・利器) の人格化。
  factor_affinity: 来歴・多視点・整合・閉ループが高い。
