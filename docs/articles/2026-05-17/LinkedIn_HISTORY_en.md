# LinkedIn Post — 2026-05-17 Project History (English)

📜 **llmesh → llove → llive — Development history and design concepts of the FullSense triad**

A timeline of the three OSS projects under the **FullSense ™** umbrella that I (Kazufumi Furuse / puruyan) develop.

🧩 **llmesh** (the eldest) — secure LLM hub
- v1.5.0 shipped (MTEngine + XbarRChart + CUSUMChart + CLI)
- Design: unify multiple LLMs (OpenAI / Anthropic / Ollama) behind MCP; embed Statistical Process Control; connect directly to industrial IoT.

🎮 **llove** (TUI HITL workbench)
- v0.2.2 first PyPI release (2026-05-09)
- F15 browser-grade rendering (Markdown / SVG / Mermaid)
- F16 multi-game LLM arena (chess / go / mahjong / poker)
- F22 Tetris demo / F23 PowerShell-compatible shell / F24 Claude Code integration
- F25 bridge layer (llove ↔ llmesh ↔ llive over MCP)
- Design: Textual-based, many demos to "win by motion."

🧠 **llive** (5 days in, today's focus) — self-evolving modular memory LLM framework
- Started 2026-05-13, v0.1.0 (Phase 1 MVR)
- 2026-05-14: v0.3 → v0.4 → v0.5 (Phases 3/4/5 same day)
- 2026-05-16: v0.6.0 (9 axes skeleton + Apache 2.0 + FullSense umbrella)
- Today: Brief API + 32 new requirements + 1,014 tests pass

🎯 **6 design pillars**:
1. Local-first is AI's natural home
2. Accountability at architecture level
3. Multi-faceted thought as structure (10 cognitive factors)
4. TRIZ to structure creativity
5. Honest disclosure as research practice
6. Build as a family (3 products, 1 worldview)

⭐ **Differentiators**:
- vs cloud LLMs (GPT / Claude / Gemini): wins on on-prem, audit ledger, disaster-mode operation
- vs agent frameworks (LangChain / CrewAI): wins on TUI HITL, formal verification, TRIZ
- vs Wolfram Alpha: wins on full OSS, LLM integration, audit log
