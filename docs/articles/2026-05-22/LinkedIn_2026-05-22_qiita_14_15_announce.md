**Qiita 連載「FullSense / llive 完全解説」のドラフトを公開し始めました。** 全 20 本の予定で、今日は #14 と #15 が走りました。

**#14 — 「HTML で見えないのに、機械では読める」**

llive (自己進化型モジュラー記憶 LLM フレームワーク) が採用している「不可視 Annotation Channel」の解説です。LLM agent が出力する HTML / Markdown に、人間には見えないけれど次の agent には読める metadata を埋める設計。RDFa / microdata の系譜にいるけれど、agent-to-agent の橋渡しに特化させた使い方です。

→ <https://qiita.com/furuse-kazufumi/items/33b70c801894b91ca826>

**#15 — 「30 年の開発経験 × Perplexity × Claude Code × TRIZ × 論文 RAG ~5 万件 = 第二の脳型スパイラル開発」**

私の開発スタイルそのものを言語化しました。タイトルがほぼ要約。Karpathy の「vibe coding」とは少し違って、harness を自分で握り続ける形を「ハーネス型バイブコーディング (HVC)」と内輪で呼んでいます。harness 型では発想力 + 経験則 + アルゴリズム理解 + AI 成長マネジメント力の 4 つがユーザー側に要る、という整理です。
