## 0. このシリーズの目的（最重要・執筆の背骨）

1. **LLM の構造を、自分で組み直して確かめた一次体験で解説する。** 教科書の図やフレームワークのブラックボックスでなく、
   「推論エンジンを自作し、公式実装と**実測して誤差ゼロで再現**した」経験を土台に、部品を一つずつ分解して見せる。
2. **誤差ゼロ再現＝理解の証明を前面に。** 自作した forward pass が公式リファレンス実装と実質誤差ゼロで一致した、という
   事実がこのシリーズ全体の信頼の核。「エンジンを組み直したら1ビットの狂いもなく同じ出力が出た。だから中身を説明できる」。
3. **記事の目的はキャリアのアピールでもある。** 著者は 25 年以上「カメラで見て、機械を動かす」計測・制御エンジニア。
   古典的な数値・光学・制御の現場力と、現代 LLM の内部実装力を橋渡しできる稀な立ち位置を、誠実に（自慢でなく事実で）示す。
4. **一般版＝腑に落とす／技術版＝数式とコードで納得する、の2本立て。** 同じ地図を二つの高度で通る。

## 2. 「誤差ゼロ再現」の一次事実（このシリーズの信頼の核・正確に引用）

出典＝自作言語モデル llcore の一次記録（`CONVERSATIONAL_LLCORE_FINDINGS.md`）。**数値はここから一字一句正確に**。

- **自作 Qwen2 デコーダ**（RMSNorm / RoPE θ=1e6 / GQA / SwiGLU / tied embeddings）を、フレームワークの中身に頼らず
  ゼロから実装。モジュール名を公式（HuggingFace transformers）に合わせ、公式の学習済み重み（state_dict）を直接ロード。
- **ゴールデンテスト（公式 `transformers.Qwen2ForCausalLM` と照合）**:
  - フルシーケンスの **logits が 2e-4 で一致**（＝浮動小数点の丸め誤差の域。実質同一）
  - **greedy 生成のトークンが完全一致**
  - **KV キャッシュ経路の出力＝フル forward の出力に一致**
- **20 ターン耐久テスト（greedy, 自作 vs 公式）**: 同一 seed・同一プロンプトで **20/20 ターン バイト単位で同一の返答**。
  - ★honest（本人が明記）: これは §1 の**決定論的な帰結**であって独立した証拠ではない。greedy は各ステップ argmax なので、
    単発 argmax 一致の当然の系。sampling（do_sample=True）/ int8 / 1.5B / 線形化版については何も言っていない。
- **メモリ最適化版ローダー（streaming loader）**: 重みを1テンソルずつ mmap から流し込む。dict ローダーと
  **ビット単位で完全一致（0.5B で max|Δ| logits = 0.0）**。
- **mmap 重みロード（別の実験）**: forward の logits が eager ロードと**完全一致（max|Δ|=0.0）**。使用 RAM 上限を
  モデルサイズ以下に絞ったプロセスでも forward 完走、**capped/uncapped の logits チェックサム完全一致**。
- **int8 ストリーミング化**（行列演算の Linear だけ int8、embedding/lm_head は fp32 維持）:
  - 0.5B: 約 2.0GB → **約 1.21GB** 常駐、会話維持
  - **1.5B: 約 5.7GB → 約 2.44GB** 常駐（fp32 スパイク無し）、会話維持
  - CPU 速度は約 0.7 tok/s（毎 forward で dequant する負荷。int8 の**速度**利得は GPU の int8 GEMM 待ち＝「良い HW ほど効く」）

**言い回しの規律**: 「誤差ゼロ」は正確には二種類ある。(a) 自作 forward vs 公式＝**logits 2e-4**（＝浮動小数点ノイズ域、
実質ゼロ）。(b) メモリ最適化版ローダー vs 素の実装＝**ビット完全一致 max|Δ|=0.0**（＝文字通りゼロ）。両方を正確に区別して書く。
「1ビットも違わず」と言えるのは (b)。(a) は「実質誤差ゼロ（浮動小数点の丸め域）」と書く。**過大表現しない**。

## 3. 会話の実結果（honest・技術版で使用）

- **0.5B を自作 forward で駆動**（~5–6 tok/s, CPU fp32）: 「日本の首都は？」→「東京都です」✓ /
  「3たす5は？」→「18」✗（0.5B は算数が弱い）。
- **1.5B を自作 forward で駆動**（~1–3 tok/s）: 「3たす5は？数字だけ」→「8」✓ / 敬語変換✓ /
  「日本で一番高い山は？」→「富士山です」✓ / しりとりは弱い。
  → **一般 Q&A・算数・指示追従で「まともに会話できる」に到達**。
- ★honest の継ぎ目: **会話の能力は Qwen の学習済み重み由来**。自作側の貢献は「中身を検査・改造できる、検証済みの
  オンプレ推論ランタイム（ブラックボックスでない）」。改造版（線形化/蒸留）が素のモデルより会話が上手いとは主張しない。

## 4. 章別の技術事実（各担当が使う正確な数値。ここに無い数値を作らない）

### 一般共通の LLM 事実（教科書レベル・timeless）
- LLM = 次トークン予測器。自己回帰（1語出す→全部読み直す→次を出す）。
- パイプライン: テキスト →①トークン化 →②埋め込み →③注意機構(attention) →④順伝播層(FFN) →（③④を N 層反復）
  →⑤出力層で語彙全体にスコア(logits)→softmax→次トークン。

### 第1章 トークンと埋め込み
- tokenizer（BPE 等）: 文を「トークン（部分語）」に分割。1トークン≠1単語（日本語は特に細かい/多い＝tokenizer fertility）。
- embedding: 各トークン ID を hidden 次元のベクトル（「意味の座標」）へ。学習で獲得。
- **tied embeddings**: 入力埋め込み行列と出力（lm_head）を共有＝語彙×hidden の巨大行列を二重に持たない（メモリ正しい設計）。
  自作実装でもこの共有を再現（小 RAM ターゲットで効く実détail）。
- 有名な直観: 「king - man + woman ≈ queen」＝ベクトル空間に意味の方向が乗る。**PCA（主成分分析）で次元を軸に分解する
  のと同じ発想**（著者の得意分野）。ただし比喩は盛らない（実際の埋め込みは非線形で完全にこの式が成り立つわけではない、と一言添える）。

### 第2章 注意機構(attention)★心臓
- self-attention: 各トークンが Query(Q)/Key(K)/Value(V) を作り、**スコア = softmax(QKᵀ/√d)**、出力 = スコア×V。
  「今の語が、過去のどの語にどれだけ注目するか」を動的に決める重み付き平均。
- **QKᵀ ＝内積類似度＝相関（テンプレートマッチング）**。著者の画像処理の相関・整合と地続き（本物の異分野ブリッジ）。
- **RoPE（Rotary Position Embedding, θ=1e6）**: 位置を「回転（周波数による位相）」として Q,K に乗せる。
  **フーリエ変換／位相の考え方と同じ土俵**（著者の得意分野。位置を周波数で符号化）。
- **GQA（Grouped-Query Attention）**: K,V のヘッドを複数の Q ヘッドで共有＝KV キャッシュのメモリ削減。
- **計算量 O(T²)・KV キャッシュ O(T)**（T=文脈長）＝「長い文章ほど急に重くなる」の正体。一般版#0 が既にこれを予告済み。
- 自作 forward が公式と 2e-4 一致した＝この attention（RoPE/GQA/KV cache 含む）を正しく組めた証拠。ここで §2 事実を使う。

### 第3章 Transformer ブロックの全体像
- 1ブロック = ①注意機構 + ②順伝播層(FFN)、各々に **残差接続(residual)** と **正規化(RMSNorm)**。
- **RMSNorm**: LayerNorm の簡略版（平均引き算なし、二乗平均で正規化）。数値的に安定・軽量。
- **FFN/SwiGLU**: 2層 MLP（ゲート付き活性化 SwiGLU）。**ここが「知識の貯蔵庫」**。
- ★知識の局在（研究知見・適度な hedge で）: 世界知識・事実は主に **FFN/MLP と出力層**に、attention は**検索・ルーティング**
  （どこを見るか）に効く、という複数系列の証拠（ROME/MEMIT/Geva らの FFN=key-value 記憶説、線形化レシピが FFN を凍結し
  attention だけ蒸留する事実）。→ 「賢さは足し算層に、注目は attention に」。**断定しすぎない**（研究の総合であって確定した唯一解ではない）。
- これを N 層（0.5B で 24 層）積む。層0や一部の初期層は attention が「本当の仕事」をしていて改造に弱い（第5章の耐性プロファイルへ伏線）。

### 第4章 学習と推論（なぜ「事前学習済み」が効くのか）
- 学習 = 大量テキストで「次トークン予測」の誤差(loss)を勾配降下で最小化。**キャリブレーション（誤差を測って補正）と同じループ**
  （著者の双腕ロボット/計測の校正と地続き）。
- **honest null（自作の一次体験・目玉）**: 自宅 CPU で**ゼロから作った文字単位 LM は会話できない**。
  最良でも 11.9M パラメータ・ppl 38（unigram 215 比）で、日本語の「文語の物真似」までで会話にならない。
  ボトルネックはアーキでなく**規模・データ・計算**。→「賢さは重みに宿る。重みは巨大な事前学習でしか入らない」。
- 定数状態 recurrent 文字 LM を青空文庫で実学習: 1.3M params, **ppl 22 vs unigram 207 = 9.4 倍**の予測力, top1 0.38。
  定数メモリで非劣化（位置 8192 まで ppl~21 平坦）だが**有効文脈 ≈ block_size(~128)**（BPTT 窓を超えて文脈を使えない honest null）。
- 推論 = 学習済み重みで次トークンを1個ずつ。temperature/top-p（sampling）は「予測分布からどう1個選ぶか」。

### 第5章 メモリと速度の壁（KVキャッシュ・量子化・線形化）— 著者の得意領域（メモリ効率）
- **KV キャッシュは文脈長に線形膨張**（実測: GPT KV T=256 4.72MB→T=2048 37.75MB ×8 線形 / 定数状態 recurrent は平坦）。
  peak RSS 実測: GPT T 256→2048 で ×2.65、recurrent/RWKV ×1.00。T=4096 で GPT ~1.67GB vs recurrent 205MB（~8倍）。
- **量子化（int8 など）**: 重みを低ビットに。int8 weight-only で**約 3.9倍（74–75%）圧縮・held-out PPL 劣化 <0.1%**。
  per-channel < per-tensor 誤差。**ビット幅の崖**: 崖の位置はモデルサイズ依存（大きいほど低ビットに頑健）。
  **3bit が PTQ の実用床、2bit は QAT 領域**（小モデルは QAT/LSQ でも 97% 保持ゲート未達）。
  ★honest 警告: **PPL だけのゲートは危険**（2bit が PPL ゲートを通っても top1 が -13.5pp 崩壊した実例）。
  →「不良を取りこぼさない」計測規律＝**capability ゲート（top-1 保持率）で fail-closed 検収**（著者の品質規律の実装）。
- **線形化（softmax attention → 定数状態 線形 attention）**: 状態 S=Σφ(k)⊗v（φ=elu+1）, per-head 状態 **O(d²) で文脈長に不変**。
  - per-head 線形状態 232,960 B/層（定数）vs softmax KV @8192 = 8,388,608 B/層（36倍）。**交差点 ≈ 227 トークン**
    （それ以下では定数状態の方が**大きい**）。
  - **「タダではない」**（本人が強く訂正）: 利得＝メモリ（長文脈のみ）、コスト＝品質（小さいが非ゼロの Δnll）＋
    短文脈の計算ペナルティ。「無料の昼食ではない、トレードだ」。
  - 層別耐性（0.5B, zero-shot, baseline ppl 68.74）: **層0 が最も非耐性**（単独線形化で ppl 160）、層 11/9/3/1 も抵抗、
    中後段はほぼゼロコスト（層22 Δ+0.0007）。累積 top-4 層で +7% ppl、12 層で破綻(167)、全24層で壊滅。
- **蒸留回復**（LoLCATs 流・小さな学習可能 feature map・恒等初期化・projection 凍結）: 最も非耐性の層を
  **held-out で 92–101% 回復**（層9 96%, 層11 101%）。**未知テキストにも汎化**。
  ★honest: 出力蒸留・1層ずつ・小 CPU モデル・PPL proxy。feature map は実は 3,584 params/層(~14.3KB)で「~4 params」は不正確
  （ただし文脈長に O(1) なので bounded-memory の主張は維持）。
- 設計指針: これらは「CPU 延命策」でなく**「良い HW ほど効く」土台**（int8→GPU で真の int8 GEMM 速度、定数状態→GPU 長文脈で
  二次メモリ壁なし）。

### 第6章 実務編（モデル選定・評価・進化・責任あるAI）— レポート性の核
- **モデル選定**: ライセンスが商用の生命線。**Apache-2.0 / MIT（商用クリーン）= Qwen2.5/Qwen3, Ministral-3, Phi-4-mini(MIT),
  SmolLM3(Apache だが日本語弱)**。**非商用の罠**: Qwen2.5-3B は Qwen Research（非商用）、Gemma は派生物伝播条項。
  日本語品質・サイズ・数学力・ライセンスの4軸で選ぶ。
- **RAG vs ファインチューニング vs 蒸留**:
  - RAG（検索拡張）＝安い・即更新・来歴が残る（大きな知識は「検索で貸す」）。
  - ファインチューニング/蒸留＝能力の移植（**教師の出力ライセンスが命**。Apache/MIT のみ。OpenAI/Anthropic 出力は ToS 禁止）。
  - 天井: 生徒 ≤ 教師。蒸留分布で良く見え他で崩れる→ cross-corpus 評価必須。
- **評価の罠（honest disclosure の核）**:
  - **winner's curse（勝者の呪い）**: たくさんの候補から最良を選ぶと、その最良値は楽観的に偏る。→ **新鮮な holdout で再評価**。
  - **PPL だけで判断しない**（第5章の 2bit の例）。top-1 保持など多軸で。
  - **文脈長スイープ**: 短文脈の proxy は長文脈のコストを過小検出（線形化の劣化は長文脈で出る）。実例: アグレッシブな
    設定で Δnll +0.45→+0.49 と**文脈長とともに単調悪化**（1.6倍 ppl 悪化）。usable band は控えめな削減（30.8% 削減で +7% ppl）。
  - proxy-v2 の思想: paired 多窓 Δnll + bootstrap CI + winner's-curse 補正 + honest_verdict チョークポイント
    （scope を「next_token_nll_proxy」に固定し、会話品質の主張を構造的に封印＝conversational_claim=None）。
  - 「異常に良い結果は内訳を疑う」＝著者の**計測規律（測定を疑う・校正する）**の LLM 版。
- **進化 × 構造（memetic NAS）**: 層別に「どの mixer（softmax/sliding-window/線形）を使うか」を探索。
  ★honest の弧: **ゼロからの GA は負ける**（二値マスク×単調予算は separable で greedy が近最適）が、
  **greedy 解を種にした memetic GA は勝つ**（3-mixer 空間で Pareto ハイパーボリューム +34.3%）。
  教訓＝進化は「万能の魔法」でなく「greedy が届かない非分離な地形での精緻化器」。
- **責任あるAI（FullSense 哲学・自慢でなく設計思想として）**: ローカル完結（データを外に出さない）、
  Approval Bus / HITL（人間の承認点）/ fail-closed / honest disclosure。「賢くなる」だけでなく「責任を持つ」スタック。
  → 計測装置の「ラインを止めない・不良を取りこぼさない」思想と同型。

## 6. 文体規約（全章共通・raptor memory 由来。厳守）

1. **構成 = ①用語ミニ辞典 → ②かみくだき（平易な全体像）→ ③詳細（数式・コード・honest 内訳）**。①②で要旨が掴め、③を読まなくても筋が通る。
2. **専門用語は初出時「日本語(English)」**（例: 注意機構(attention)、埋め込み(embedding)、量子化(quantization)）。
   4文字以下の略語はセッション初出1回だけ展開（例: LLM(Large Language Model)、KV(Key-Value)）。以降は日本語/略語で通す。
   一般的な英単語（honest, clean 等）を日本語文に生で混ぜない（→ 正直な/素直な 等に）。コード識別子はそのまま。
3. **かみくだきは中学理科・数学を超える前提を要求しない**が、**「中学生でもわかる」等の見下し枕詞は絶対に書かない**（内部基準であって読者に宣言しない）。
4. **異分野フレーバー**（生物・漫画・アニメ・そして著者の計測現場）で堅さを抜く。ただし**比喩のために事実を盛らない**。
5. **honest disclosure を前面に**: 失敗（文字LM会話不能・2bit崩壊・GAが負ける）を消さず教訓に。留保・条件を必ず併記。
   「異常に良い結果は内訳を疑う」。数値は §2–§4 の実測値のみ。**無い数値を作らない**。
6. **仁の倫理ゲート**（発信物の作法）: 非暴力（敵/殴る/潰す等の攻撃比喩を使わない→反証/検証へ）、非脚色（来歴・主語を事実通り）、
   非煽情（刺激で釣らない、中身で読ませる）、思いやり（読者・競合を貶めない）。
7. **謙虚の作法**: 自賛語（世界初/最強/圧勝/革命的）禁止。成果は事実で淡々と、評価は読者に委ねる。確信あることは堂々と、
   不確実は控えめに（両方使い分ける）。
8. **「次回に続く」で引く**（漫画のヒキ）。各章末に次章への伏線・問い。前章の引きを次章冒頭で受ける。論文と逆。
9. **記事は読者へのプレゼント**: 読者が持ち帰れるもの（気づき・道具・人に話したくなる話題）を必ず1つ。
10. **語呂/連想**（例: 「次の一語オタク」）で retention を上げる。
11. **バイライン**: 各記事冒頭に「著者: 古瀬 和文（ぷるやん）」。
12. **公開資料の禁止事項**: ローカルパス（D:/... 等）を書かない。所属企業の固有名詞を書かない。ファイル名/内部変数名を晒さない
    （概念で語る。「自作の推論ランタイム」等）。
13. **Qiita 仕様**: tags はスペース区切り・最大5件。長尺歓迎（技術版は各 15,000–30,000 字級を目標、削らない）。
    画像は raw.githubusercontent 絶対 URL・静的状態が見える SVG（詳細は IMAGE_PROMPTS）。ただし**画像生成自体は他 AI に委譲**するので、
    本文には画像の**プレースホルダ（キャプション + 生成意図コメント）と Mermaid 図**を置く。
14. **コードは自作の実コードを晒さない**。**クリーンな教育用の擬似コード/最小例**を書く（IP 保護 + 教育効果）。PyTorch 風で可読に。
