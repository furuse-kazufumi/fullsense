# GPU付きPCの納品を待つ間に、CPUでできることを全部やってみた

> RTX 5090 付きの PC を注文したのですが、ケース入荷待ちで納品がまだです。重い学習は GPU が来てから、と決めているので、いまは**ノート PC の CPU でできる小ネタ**をつまみ食いしています。この記事は、その待ち時間にやった 3 つ ── **歩行ロボの進化**・**LLM のマージ**・**性格ダイヤル** ── を、軽いノリでまとめたものです。
>
> ひとつだけ先に言っておくと、**全部が「うまくいった話」ではありません。** むしろ 2 つは「正直つらい話(null)」です。でも個人的には、負けの内訳がちゃんと見えたほうが次に効くと思っているので、負けも含めて書きます(この方針を honest disclosure と呼んでいます)。気楽に読んでください。

## 0. 3行でいうと

- **歩行ロボの進化**: 「どの親を選ぶか」を変えたら、多様性は減るのに質は上がる、という条件付きの相乗を見つけました(ただし代償あり)。
- **LLM のマージ**: ロボの進化エンジンを流用して LLM のマージを探索しました。最初は「勝った!」と思ったのに、**ベースラインを公平にした瞬間に優位が消えました**(偽陽性でした)。
- **性格ダイヤル**: 重みではなく「活性(内部信号)」をいじる手法は、**可逆・再訓練なし**であっさり効きました。感情や口調をダイヤルで回せます。

## 1. 歩行ロボの進化 ― 「親の選び方」で質が上がる話

まず物理シミュレータ(MuJoCo)の中で、四足・二足ロボットの**歩き方**を進化させています。ROS 2 の制御スタックにも載る形で作っていて、ここは「GPU が来たら大規模並列にする」ための地ならしです。

進化には **MAP-Elites**(品質と多様性を同時に保つ手法)を使っています。「速く歩く個体」だけでなく「跳ねる」「すり足」など**多様な歩き方**を、格子に保存し続けます。

ここで実験したのが、**親の選び方(選択圧)**です。ふつうは性能の良い個体を親にしますが、**lexicase 選択**という「複数の評価軸をランダムな順で 1 つずつ通して勝ち残らせる」方式を混ぜてみました。結果:

- **質(objective のフロンティア)は上がる。** 複数の目的を同時に満たす個体が出やすくなる。
- **でも多様性(coverage)はむしろ減る。** 一部の勝ちパターンに寄ってしまう。
- つまり **条件付きの相乗**でした。「良いとこ取り」ではなく「あちらを立てればこちらが立たず」。

正直に言うと、最初は「多様性も質も両方上がった!」と思ったのですが、**多様性の指標をちゃんと clean に測り直したら、多様性はむしろ下がっていました**。ここでも自己修正が入っています。最終的には、coverage を守る仕組みを足した hybrid でトレードオフを緩めるところまで持っていきました。

> ちなみにこの過程で、シミュレータ + ROS 制御プラグインの**バグ**(名前のないアクチュエータで NULL 参照 → クラッシュ)を踏み抜いて特定しました。デバッグの副産物としては悪くないおまけです。

## 2. LLM って混ぜられるの? ― 理論はシンプル、実験は正直つらい

次は毛色を変えて LLM です。「日本語が得意なモデル」と「コードが得意なモデル」を 1 つに**融合**できたら省メモリで嬉しいよね、という素朴な動機。

### 2-1. まず理論(ここは意外とスッキリしている)

- **同じ土台(base)から派生した兄弟モデル**同士なら、重みの足し算・平均でマージできます(Model Soups / Task Arithmetic / TIES / DARE などの系譜)。
- でも**「全く別」のモデル**(構造もトークナイザも違う)は、**重みでは原理的に混ざりません**。各モデルの重みは別々の座標系で意味を持っていて、マス目を重ねて平均しても意味をなさないからです(Git Re-Basin の教訓)。
- 「全く別」を 1 つにできるのは、重みではなく**振る舞いを真似させる蒸留(distillation)**だけ。

理屈はこれで終わりです。問題はここから。**「同じ土台の兄弟なら混ざる」なら、混ぜ方を賢く探索すれば得するのでは?** ── そう思って、§1 の**ロボット進化エンジンをそのまま流用**しました。歩き方の代わりに「マージのレシピ」を進化させる。名付けて QD-of-merges。

### 2-2. 実験:「勝った!」と思ったら偽陽性だった

同じ土台の兄弟(素の言語モデル ↔ 指示追従モデル)を、**層ごとに違う強さで**混ぜる配分を進化探索しました。評価は held-out の perplexity(次の単語をどれだけ当てられるか)。対抗馬は「全層を同じ強さで混ぜる」素朴なマージです。

- 進化探索の最良: バランス指標 **0.797**
- 素朴なマージ(**7 点**で測定)の最良: **0.727**

「進化が勝った! 層ごとに配分する自由度が効いた!」── この時は本気でそう思いました。

でも一拍おいて、**対抗馬の 7 点、粗すぎない?** と疑いました。素朴なマージの曲線をよく見ると、鋭い谷を持っていて、7 点だと谷の底を跨いで見落とす可能性がある。そこで **51 点**に細かくして測り直すと:

- 素朴なマージ(**51 点**)の最良: **0.838**

**進化探索(0.797)は、細かくした素朴なマージ(0.838)に負けました。** 最初の「勝ち」は、比較相手が粗かっただけの**偽陽性**だったのです。

### 2-3. 「ちゃんとした」手法でも、素朴な足し算に負けた

「単一の差分が 1 次元だからでは。2 つのエキスパートを混ぜれば話が変わる」と思い、今度は自分で **2 人目のエキスパート**を用意しました(素の base を Python コードで軽く微調整して "コード担当" を作成。コードの perplexity が半減しました)。指示担当 × コード担当を、衝突を賢く解消するという **TIES** で混ぜます。

結果は、またしても正直つらいものでした:

- **素朴な足し算(task arithmetic): 0.746** ← 勝者
- TIES(いろいろ調整): **0.000 〜 0.662** ← 全敗

TIES は「意見が食い違う所は多数決で片方を採る」手法です。でも今回の 2 人は**相補的**でした。指示能力とコード能力は**両方あってこそ**なので、多数決で片方を捨てる TIES が、かえって邪魔をした。**TIES は本当に衝突を解消したい時にしか効かない**、という当たり前だけど大事な確認でした。

というわけで、LLM のマージは **2 連続で honest null**。凝った探索は、この設定(小さいモデル・相補的なエキスパート)では割に合いませんでした。**手法が良く見える最大の理由は、しばしば「比較相手が弱いこと」です。**

## 3. でも「場所」を変えたら、あっさり効いた ― 性格ダイヤル

重みを混ぜる話が全部 null だったので、**別の場所**を触りました。重みではなく、推論中の**活性(activation、内部の信号)**です。

やり方はシンプル(ActAdd / RepE という手法):

1. 対照的なプロンプトを用意する。「I love this, wonderful and…」(ポジ)と「I hate this, terrible and…」(ネガ)。
2. ある層での内部信号の**差ベクトル**を作る。これが「ポジ ↔ ネガ」の方向。
3. 推論中、その層に **α 倍した差ベクトルを足す**。α を大きくするとポジ寄り、負にするとネガ寄り。**足すのをやめれば完全に元通り**(重みは一切変えていない)。

これが、**あっさり効きました**。感情の出やすさが α に沿ってきれいに動き、フックを外すと**厳密に元の値に戻る**(差ゼロ=完全に可逆)。生成も、中立の文が α を上げると "filled with peace, and a beautiful celebration…" のように、ちゃんとポジティブになります。

さらに、これは**複数の軸を合成**できます。「感情」と「口調(形式的 ↔ くだけた)」を別々のダイヤルとして作り、同時に回すと:

| 設定 | 感情の出やすさ | 形式度 |
|---|---|---|
| ぜんぶ 0 | +2.8 | −3.0 |
| 感情 + | **+5.3** | −3.0(ほぼ不変) |
| 形式度 + | +2.8(ほぼ不変) | **+5.1** |
| ポジ + 形式的 | +4.4 | +6.1 |
| ポジ + くだけた | +3.1 | −7.4 |

**各ダイヤルは主に自分の軸だけを動かし、混ぜても加算的に効きます。** まさに「性格ダイヤル」。正直な限界もあって、今回のモデルは 1.35 億パラメータと小さいので、ダイヤルを回しすぎると文が壊れます(過剰操舵)。それでも、可逆で再訓練いらずというのは扱いやすい。ゆくゆくは自作の対話ツールに載せて、その場で性格を回せるようにしたいと思っています。

## 4. おわりに ― GPU が来たら本気を出す

まとめると、GPU 待ちの CPU 期間にやったのは:

- **歩行進化**: 親の選び方で質は上がるが多様性は減る、という条件付き相乗(自己修正込み)。
- **LLM マージ**: 進化エンジンの流用は動いたが、賢い探索は素朴なベースラインに 2 連敗(偽陽性の発見つき)。
- **性格ダイヤル**: 重みではなく活性を触ると、可逆・再訓練なしで人格を回せた。

派手な成果はありませんが、**「どこで負けたか」の地図**はだいぶ埋まりました。個人的には、これがいちばんの収穫だと思っています。異常に良い結果が出たら、勝った気になる前に**まず比較相手を疑う** ── これは GPU があってもなくても効く教訓でした。

GPU が届いたら、歩行進化を数千環境の並列にしたり、まともな蒸留を回したりする予定です。その話はまた別途。ここまで、待ち時間の小ネタにお付き合いいただきありがとうございました。

> 使った主な手法(一次情報): MAP-Elites (arXiv:1504.04909) / lexicase 選択 / Task Arithmetic (2212.04089) / TIES (2306.01708) / DARE (2311.03099) / Git Re-Basin (2209.04836) / ActAdd (2308.10248) / RepE (2310.01405)。実験は全て CPU・小型モデル・単一 seed の範囲の話で、大きいモデルでは別の結末があり得ます。
