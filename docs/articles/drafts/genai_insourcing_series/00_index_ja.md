# 生成AIを内製する 〜自宅から世界モデルへ〜(連載トップ)

> 連載インデックス(全9章・第1回〜第9回)。各章の入口と読む順、そして「どこまで正直か」をここに集約します。

## はじめに — なぜ「連載トップ」から読むと得か

「生成AIを内製したい」。そう口にした瞬間、多くの方の頭には *ゼロからモデルを訓練する自分* の姿が浮かびます。GPU をうならせ、乱数から立ち上がった重みが少しずつ言葉を覚えていく——ロマンのある絵です。ですが、そのロマンにいきなり飛び込むと、たいていは数千 GPU 時間と数十万ドルの請求書という現実に頭を打ちます。

この連載「**生成AIを内製する 〜自宅から世界モデルへ〜**」は、その名のとおり **「自宅の机の上にある PC 一台から、少しずつ世界モデルへ手を伸ばしていく」道のり**を、正直な桁感つきで一緒に歩く企画です。派手な成功譚ではなく、**どこにお金と時間が消えるのか**、**どこは自宅で本当に作れて、どこは借りても割に合わないのか**を、フェルミ推定(概算による桁の見積もり)で毎回はっきりさせていきます。

このトップページは、全9章の**地図の地図**です。どの章が何に答えるのか、どこから読むと迷わないのか、そして「この連載がどこで嘘をつかないと約束しているのか」を、最初に一望できるようにしました。気になる章から飛んでも構いませんが、全体は **1本の川**として設計しています——上流の「どこまで作るか」という意思決定から、下流の「自宅から世界モデルへ」という到達点まで、水は一方向に流れます。

## この連載の狙い

一言でいえば、**「生成AIを"自分の手で"作る道筋を、最終的な世界モデル構築まで一続きで描く」**ことです。

世の中には「RAG の作り方」「LoRA の回し方」「世界モデルの解説」が個別には数多くあります。しかし、それらを **1枚の地図**の上に置き、「自宅の GPU 1 枚という制約から見たとき、どれが地続きで、どこに崖があるのか」を通しで描いた資料は多くありません。本連載は次の3つを最後まで手放しません。

1. **一本の軸で並べる。** 「握力のダイヤル」(重みにどこまで触れるか)と「6ND 則」(学習コストの桁)という共通の物差しで、RAG から from-scratch、さらに世界モデルまでを同じ地図に載せます。
2. **桁で語る。** すべての可否判断を、気合いや印象ではなく **VRAM(机の広さ)と TFLOPS(火力)、そしてスケーリング則**という物理法則に還元します。
3. **正直に切り分ける。** 「自宅で作れるもの」と「作れないもの」を誇張せずに分け、作れないと分かったものは潔く手放して、作れるものへ資源を集中します。

## 誰に向く連載か

- **生成AIを「使う側」から「作る側」へ半歩踏み出したい**エンジニア・研究者・意思決定者。
- 「独自 LLM が欲しい」「世界モデルを試したい」と言われて、**予算と工期の桁を先に見積もりたい**立場の方。
- 自宅や小規模ラボの **GPU 1〜数枚**という現実的な制約の中で、何が地に足のついた計画かを知りたい方。
- 逆に、**すでに大規模クラスタを持っている方には物足りない**かもしれません。本連載の主戦場はあくまで「自宅の机の上」です。

数式は出てきますが、各章は **用語ミニ辞書 → かみくだき(比喩) → 詳細・原理**の順で組み立ててあるので、数式が苦手でも比喩の層で全体像はつかめるように書いています。

## この連載の共通語彙(ミニ辞書)

各章で都度定義しますが、全体を貫く言葉だけ先に置いておきます。迷ったらここへ戻ってきてください。

- **内製(in-house)**: モデルの重みや振る舞いを「自分の手に握る」度合い。*0か1* ではなく **連続体(グラデーション)**として扱います。
- **フェルミ推定(Fermi estimation)**: 分解・仮定・出所を明示したうえで、答えを **桁(オーダー)**で概算する方法。本連載の数値はすべてこの精度です(有効数字ではなく桁感)。
- **6ND 則($C\approx6ND$)**: 学習に要する計算量 $C$(FLOPs)が、パラメータ数 $N$ と学習トークン数 $D$ の積のおよそ 6 倍になる、という経験則。コストの桁を暗算する主力の道具です。
- **スケーリング則(scaling laws)/ Chinchilla**: 損失が $N,D,C$ の冪乗で決まるという経験則(Kaplan et al. 2020 / Hoffmann et al. 2022)。「計算量が決まれば最適なモデルサイズも決まる」という天井を与えます。
- **世界モデル(world model)**: 「行動で条件づけた、時間発展する生成器」。単に *尤もらしい続きの動画* を流す装置ではなく、**こちらの行動で未来が分岐する**内部シミュレータを指します(第6回で厳密化)。
- **honest disclosure(正直な開示)**: 異常に良い結果は内訳を疑い、不確実な点は「(要検証)」と明記し、自宅で作れる/作れないを誇張せずに切り分ける、本連載の基本姿勢。

## 全体の構成 — 3幕と読む順

各章は独立しても読めますが、**第1回から順に読むのが最短**です。全体は次の3幕に分かれます。

- **第1幕(第1〜3回)内製の意思決定とエンジン** — どこまで自分で作るか、律速はどこか、学習をどう回すか。
- **第2幕(第4〜5回)フェルミの価格表** — GPU 1 枚で「何が」「どこまで」作れるかを桁で叩き、自宅内製の**価格表**を手に入れます。
- **第3幕(第6〜9回)世界モデルへ** — その価格表を握ったまま、生成AIの部品を世界モデルへ組み替え、実行ロードマップに畳みます。

## 各章への案内

各章の**引き(hook)**を1〜2文に要約し、「この回で手に入るもの」を添えます。番号順が推奨の読み順です。

### 第1幕 — 内製の意思決定とエンジン

#### 第1回 どこまで自分で作るか — from-scratch から RAG までの地図
「ゼロから訓練する自分」というロマンは、たいてい数千 GPU 時間と数十万ドルの請求書に頭を打ちます。だからまず地図を広げ、お金と時間がどこに消えるのかを桁で先読みします。**5段階の内製(RAG / SFT+LoRA / 蒸留 / 継続事前学習 / from-scratch)を「握力のダイヤル」という一本の軸に並べ、6ND 則で各段のコストを暗算できるようにする**、連載の起点です。

#### 第2回 本当のボトルネックはデータ — 内製の土台
多くの人がモデル構造の選定へ進もうとする瞬間、実務家はほぼ例外なく逆説に行き着きます——「律速はモデルではなくデータパイプラインである」。**CommonCrawl 収集・品質フィルタ・MinHash/LSH 重複除去・混合比・トークナイザまでを精錬所の比喩で第一原理から分解し、自宅で何トークンまで作れるかを正直に切り分ける**、土台の回です。

#### 第3回 GPUで学習を回す — フレームワークと分散の実務
設計図が美しくても、燃料系・冷却系を組み切れなければ車は一メートルも走りません。**`CUDA out of memory` と `loss: nan` という2つの壁の正体**を、16 byte/param のメモリ内訳から解き明かし、メモリ削減(混合精度→勾配チェックポイント→QLoRA)と4軸の分散並列(DP/TP/PP/CP)、発散させない安定化までを桁で扱います。「事前学習は個人の域外だが、微調整と小規模事前学習は自宅で回る」という分水嶺を示します。

### 第2幕 — フェルミの価格表

#### 第4回 1枚のGPUで何が作れるか — フェルミ推定で桁を掴む
「◯◯B のモデルをうちの GPU でゼロから」——その可否は気合いではなく、**1枚のカードに刻まれた2つの上限(VRAM と TFLOPS)**でほぼ機械的に決まります。RTX 5090(32GB)を素材に、from-scratch で作れるサイズ、推論・フル微調整・QLoRA の上限、自宅とクラウドの損益分岐を、仮定と出所つきのフェルミ推定で桁ごと掴みます。末尾に「封筒の裏3分レシピ」つき。

#### 第5回 自宅で作れるもの・作れないもの
自宅内製の天井を決めるのは演算速度ではなく、**メモリとスケーリング則の2つ**——しかも電卓で叩けます。6ND 則・Chinchilla 則・VRAM 会計を「価格表」として叩き、**作れるもの(nanoGPT 再現・小型 SLM・蒸留・QLoRA 特化・DreamerV3 級の特化世界モデル)と作れないもの(フロンティア base・汎用動画世界モデル)を物理法則で正直に切り分けます**。第2幕の総まとめであり、第3幕への切符です。

### 第3幕 — 世界モデルへ

#### 第6回 世界モデルとは何か — Ha & Schmidhuber から Genie まで
「動画が作れること」と「世界が分かっていること」は別物です。**Sora 論争を自分の頭で裁定できるようになる補助線**を、2018年の World Models から2025年の Genie 3 / V-JEPA 2 までの系譜(arXiv 番号つき一次情報)と4つの分類軸から手に入れ、最後に 6ND 則で「自宅で作れる状態ベース系/作れない動画基盤系」を桁で切り分けます。

#### 第7回 世界モデルの作り方 — Dreamer / JEPA / 動画世界モデル
「AI が動画を作った」デモを世界モデルだと思ったなら、半分取り逃がしています。**尤もらしい続きを流す「映画」と、こちらの行動で未来が分岐する「頭の中のゲームエンジン」の違い**こそが、鑑賞ではなく制御に効く本質です。世界モデルを5部品の配線図に分解し、Dreamer/JEPA/拡散/トークン自己回帰の4系統を長短つきで比較。制御向けの小〜中規模は5090一枚・数日、foundation 級はクラスタ案件、と桁で線を引きます。

#### 第8回 生成AIの部品が世界モデルになる — 橋渡し
「これらの部品は、既に世界モデルを内製するための部品箱である」。**自己注意・トークン化・拡散/flow・事前学習・RLHF/GRPO・推論最適化といった生成AIの部品が、そのまま世界モデルの部品箱になる**という一本の太い線を引く回です。部品対応表・3系統+計画・2本のフェルミ推定(訓練~1日、1分の想像で KV cache ~34GB)で、gaitlab / llcore / llive への具体接続まで橋を架けます。

#### 第9回(最終回)自宅から世界モデルへ — 実行ロードマップ
Genie 3 や Sora を見て「あれを自宅で」と膨らんだ夢が、GPU 1枚・VRAM 32GB の壁に激突して企画ごと諦める——その夢を**「桁で分解し、踏める段に畳み直す地図」**として、S0→S4 の5段の階段にまとめる最終回です。汎用フロンティアは燃料の桁差で不可能と認めたうえで「特定環境の実用世界モデル」へ土俵を変え、MuJoCo が CPU で回る利点を使い、GPU 着荷前から踏める第一手までを示します。

## 正直な但し書き(honest disclosure)

本連載を読み進める前に、**この連載が何を約束し、何を約束しないか**をはっきりさせておきます。

- **数値はすべてフェルミの「桁感」です。** 有効数字ではありません。各数値には可能な限り「分解・仮定・出所」を併記していますが、環境(実効 TFLOPS・電力・データ品質・実装効率)で容易に2倍・半分は動きます。断定ではなく **オーダーの目安**として受け取ってください。不確実な箇所は「(要検証)」と明記します。
- **フロンティア級は自宅では作れません。** これは気合いの問題ではなく燃料(GPU 年)の桁の問題です。汎用の巨大 base LLM や汎用動画世界モデル(Genie 3 級)は、自宅の GPU 1〜数枚とは**数百〜数千 GPU 年**の差があります。本連載はここで見栄を張らず、**「特定環境の実用世界モデル」へ土俵を変える**戦略を一貫して採ります。「作れない」を正直に手放した瞬間に、作れるものへ資源を集中できる——これが連載の背骨です。
- **この連載トップおよび各章は、まだ図が入っていません。** 本文中の「図: ○○」はすべて**プレースホルダ(意図を書いたキャプションのみ)**です。publish 前に、少なくとも「連載の3幕構成」「握力のダイヤル」「6ND 則の桁」「世界モデル5部品の配線図」「S0→S4 の階段」あたりは実図の用意が必要です(下記の TODO 参照)。
- **章番号・公開順は暫定です。** 現状は draft 段階であり、公開時にプラットフォーム上の連番採番と相互リンク(前後の章への導線)を張り直す必要があります。
- **プロジェクト固有の話(gaitlab / llcore / llive / FullSense)は題材であって宣伝ではありません。** 自宅で手を動かす具体例として登場しますが、読者が自分の環境に置き換えられる形で書くことを優先します。

### 密度と姿勢

- 記事は**読者へのプレゼント**。career-grade の密度を保ち、**薄い記事は禁止**。「20分読む価値」を基準にする。**長いことは(冗長でない限り)良いこと**。
- **敬称・丁寧体は必須**(「〜します」「〜でしょう」等)。読者を見下す表現(「中学生でも分かる」等)は**本文に書かない**——かみくだきは比喩と図式で行い、態度で示さない。
- 語調は**謙虚に、確信度で強弱を付ける**(断定と推測を区別し、自慢を避ける)。**語呂・連想**は記憶定着に歓迎。

### 章構成(6ステップ)

各章はこの順序を骨子にする。

1. **引きのある導入** — なぜ読むと得か、前章からどうつながるか(cliffhanger の回収)。
2. **用語ミニ辞書** — 「**日本語(English)**」表記。略語は初出で必ず展開(例: 検索拡張生成(Retrieval-Augmented Generation, RAG))。
3. **かみくだき** — 比喩・図式で直感を与える(見下し表現は使わない)。
4. **詳細・原理** — 数式・具体ツール名・データセット名・フェルミの桁。**厳密さを削らない**。
5. **正直な限界(honest disclosure)** — 誇張しない。不確実は「(要検証)」。自宅で作れる/作れないを正直に。
6. **次回予告(cliffhanger)** — 次章へ引く一段を必ず置く。

### 数値・図・体裁

- **数値・コストはフェルミ推定として提示**し、仮定・出所を併記する(断定ではなく桁感)。
- **図は「図の意図」を書いたキャプションのプレースホルダのみ**(例: 「図: 6ND 則でコストの桁を掴む」)。実画像の埋め込みや外部URLは書かない——画像は別途あとで用意する。
- Markdown で書く。見出しは `##` / `###`。数式は `$...$` / `$$...$$`。

### 禁止事項(公開資料の規律)

- **ローカル絶対パス(`D:\…` / `C:\…` / `/Users/…` 等)は絶対に書かない。**
- **外部URL・実画像は本文に書かない**(下書き段階のため)。
