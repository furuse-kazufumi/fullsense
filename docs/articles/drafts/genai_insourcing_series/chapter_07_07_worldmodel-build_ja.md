# 第7回 世界モデルの作り方 — Dreamer / JEPA / 動画世界モデル

> 連載「**生成AIを内製する 〜自宅から世界モデルへ〜**」の第7回です。この連載はずっと、副題の「**世界モデルへ**」という一点に向かって歩いてきました。第4回で「1枚のGPUで何が作れるか」を桁で掴み、その後の回で公開重みからの蒸留や学習データの整え方を積み上げてきました。今回はいよいよ、その道具立てを一枚の配線図に組み上げます——**自分の手で、世界モデルをどこから作り始め、どこで詰まり、自宅の RTX 5090 一枚で何が作れて何が作れないのか**。第一原理とフェルミ推定で、正直に切り分けていきます。

## 7.0 導入 — 「映画」ではなく「ゲームエンジン」を作る

生成AIのニュースを追っていると、「AIが動画を作った」「まるでゲームのような映像を無限に生成する」という見出しをよく目にします。SoraやGenieの、あの息を呑むデモです。あれを見て「世界モデルってこういう映像生成のことか」と思ったなら——半分正しくて、半分は決定的に取り逃がしています。

違いを一言で言えば、**「映画」か「ゲームエンジン」か**です。

どれだけ精緻でも、こちらの操作を受け付けず、ただ尤もらしい続きを流し続けるものは「映画」です。美しいけれど、あなたはそこに介入できません。一方、**コントローラを握って左に倒したらキャラが左に動く**——こちらの行動$a_t$で未来の分岐が変わるもの。それが世界モデル、すなわち「頭の中のゲームエンジン」です。

この違いは趣味の問題ではありません。世界モデルの最大の使いみちは、鑑賞ではなく**制御**だからです。エージェントが「この行動をとったら次はこうなる」を頭の中でシミュレートできれば、実世界で失敗を重ねる前に、**想像の中で**方策を磨けます。ロボットが転ぶ前に転び方を学び、車がぶつかる前にぶつからない運転を学ぶ。これが世界モデルベース強化学習(MBRL)の核であり、この連載が最後に辿り着きたかった場所です。

そして朗報があります。**この「ゲームエンジン」の小型版は、自宅の1枚のGPUで本当に作れます**。foundation級の汎用映像世界モデルは桁で無理ですが(後述します、正直に)、制御に効く小さな世界モデルは射程内です。本章を読み終えると、あなたは「世界モデルは5つの部品の配線図だ」と分解でき、そのどのブロックを自分の手で埋めればいいかを、封筒の裏で設計できるようになります。それが今回の"得"です。

## 7.1 用語ミニ辞書

本章に出てくる言葉を、先に「日本語(English)」で並べておきます。眺めるだけで結構です。後半で使いながら育てます。

- **世界モデル(world model)**: エージェントが観測と行動から環境の**内部シミュレータ**を学び、「この行動をとったら次はこうなる」を予測する生成モデル。制御・計画・想像上の学習に使います。
- **潜在ダイナミクス(latent dynamics)**: 生ピクセルではなく、圧縮された**潜在状態** $z_t$ の上で「次状態」を予測する遷移モデル。世界モデルの心臓部です。
- **RSSM = 再帰状態空間モデル(Recurrent State-Space Model)**: 決定的な再帰隠れ状態 $h_t$ と確率的な潜在 $z_t$ を分けて持つ潜在ダイナミクス。Dreamer系の中核です。
- **VAE = 変分オートエンコーダ(Variational Autoencoder)** / **VQ = ベクトル量子化(Vector Quantization)**: 観測を連続 or 離散の潜在に圧縮するエンコーダ手法。VQ は潜在を「語彙(コードブック)」に落とし、トークンの自己回帰を可能にします。
- **JEPA = 結合埋め込み予測アーキテクチャ(Joint-Embedding Predictive Architecture)**: 生成(再構成)をせず、**特徴空間で**未来や欠損を予測する自己教師あり法。
- **ST-transformer = 時空間トランスフォーマ(Spatio-Temporal Transformer)**: 空間方向と時間方向の注意を分離して計算量を抑えた、動画向けの Transformer。
- **行動条件付け(action conditioning)**: 遷移予測に行動 $a_t$ を入力し「行動を効かせる」機構。世界モデルが**制御可能**であるための必須要件です。
- **MBRL = モデルベース強化学習(Model-Based RL)**: 学習した世界モデルの中で(=想像の中で)方策を学ぶ/計画する枠組み。世界モデルの最大の下流応用です。
- **ロールアウト(rollout)**: 世界モデルを自己回帰で何ステップも走らせて未来系列を生成すること。長期ロールアウトの**ドリフト(誤差蓄積)**が、内製最大の難所になります。

## 7.2 かみくだき — 世界モデルは「5部品の配線図」

いきなり数式に入る前に、直感の絵を一枚だけ描かせてください。世界モデルを「自作ゲームエンジンの配線図」だと思うと、全体像がすっきりします。要点は先に2つだけ押さえます。

**その1: 生ピクセルで考えない。** $256\times256\times3\approx 20$ 万次元の画像を、毎ステップ生成しながら計画するのは高コストです。人間だって、目を閉じて「この角を左に曲がったら何が見えるか」を思い浮かべるとき、網膜の全画素を再現しているわけではありません。**要約された景色**を思い描いているだけです。世界モデルも同じで、まず観測を数百次元の**潜在** $z_t$ に畳み、その狭い空間で「次はこうなる」を回します。潜在で100ステップ想像してからデコードする方が、ピクセルで100ステップ回すより桁で安いのです。

**その2: 行動という舵を必ず握らせる。** 7.0で述べた「映画 vs ゲームエンジン」がここに効きます。ただ続きを当てるだけのモデルは映画です。$a_t$ を入れて、行動を変えたら未来が分岐して初めて、計画や強化学習に使える**世界モデル**になります。

この2つを踏まえると、世界モデルは次の5個の部品の配線図として描けます(詳細は7.3)。

> **観測を畳む(①エンコーダ) → 潜在で先を読む(②ダイナミクス) → 必要なら戻す/報酬を読む(③デコーダ・⑤報酬) ← 舵を差し込む(④行動条件付け)**

図: 世界モデルの5部品配線図。観測$o_t$を潜在$z_t$へ畳み、行動$a_t$を舵として差し込み、潜在空間で$z_{t+1}$を予測、必要に応じて観測へ戻すか報酬を読む——の流れを一枚で。

面白いのは、この連載の読者が進めているOSSが、**それぞれこの配線図の別々のブロックを埋める作業**として一枚に載ることです。llcore の「効率アーキで潜在遷移を安く回す」は②のダイナミクスを軽くする話。gaitlab の「MuJoCoで行動条件付きデータを無限に採る」は④に只で舵を供給する話。llive(自己進化記憶)はこの配線図の**外側**にメタ制御として被さる話。バラバラに見えた研究が、世界モデルという一枚の図の上で役割分担として繋がるのです。

## 7.3 原理・実務 — 世界モデルを構成する5部品

では配線図を1ブロックずつ開けていきます。内製とは、結局「各ブロックを何で実装し、どう繋ぐか」を決める作業に他なりません。

### 7.3.1 (1) 観測エンコーダ — 選択が下流すべてを規定する

生観測 $o_t$(画像・固有受容感覚・LiDAR など)を潜在 $z_t$ に圧縮します。

$$z_t = q_\phi(o_t)$$

実装は CNN/ViT の後段に、(a) 連続VAE、(b) 離散VQ、(c) 潜在のみ(JEPA、デコーダ無し)のいずれかを置きます。ここが**最初の分岐点**で、しかも一番後戻りしにくい選択です。というのも、エンコーダの型が下流のダイナミクスを規定してしまうからです。

- **VQ(離散)**を選べば、潜在が「語彙」になるので、そのままトークンの自己回帰(Genie / IRIS 系)へ自然に繋がります。
- **連続**なら、RSSM(Dreamer系)や拡散(DIAMOND系)へ流れます。

基礎は `VQ-VAE`(van den Oord et al., 2017)。「観測を語彙に落とす」という発想がここで生まれました。

### 7.3.2 (2) 潜在ダイナミクス — 世界モデルの心臓

配線図の心臓です。潜在の「次」を予測します。

$$z_{t+1}\sim p_\theta(z_{t+1}\mid z_{\le t}, a_{\le t})$$

実装系統は大きく3つ。ここは正直、**どれを選ぶかで研究の性格が決まる**ので、長所を並べておきます。

- **再帰(RSSM)**: 決定的経路 $h_t=f(h_{t-1},z_{t-1},a_{t-1})$ に、確率的潜在 $z_t\sim p(z_t\mid h_t)$ を組み合わせます。`PlaNet`(Hafner et al., 2019)/ `Dreamer`(Hafner et al., 2020)。**少パラメータで長期一貫性が出やすく、単一GPUで回る**のが最大の美点。自宅内製の第一候補です。
- **トークン自己回帰(autoregressive over tokens)**: VQトークン列を Transformer で次トークン予測。`IRIS: Transformers are Sample-Efficient World Models`(Micheli et al., 2023)、`Genie`(Bruce et al., 2024)。表現力が高く動画スケールに乗りますが、トークナイザ品質に律速されます。
- **拡散(diffusion)**: 次フレーム(or 次潜在)を条件付き拡散で生成。`DIAMOND: Diffusion for World Modeling`(Alonso et al., 2024)、`GameNGen`(Valevski et al., 2024)。視覚的細部が残る一方、1ステップに複数回のデノイズが要り、リアルタイム化と長期安定が高コストです。

### 7.3.3 (3) デコーダ or 潜在予測ヘッド — 「戻すか、戻さないか」

潜在を観測に**戻す**(再構成ベース)か、戻さず**潜在のまま**次を予測する(JEPA / 価値等価)か。ここは思想の分かれ道です。

- **戻す派(再構成)**: 人が見て検証でき、報酬も画像から取れます。ただし、制御に無関係な背景ディテールの再構成に、モデル容量を食われるのが弱点。
- **戻さない派(潜在予測)**: 制御に効く量だけを表現できて効率的ですが、潜在が一点に潰れる**崩壊(collapse)**を避ける工夫が要ります。`I-JEPA`(Assran et al., 2023)/ `V-JEPA`(Bardes et al., 2024)、思想の出典は `A Path Towards Autonomous Machine Intelligence`(LeCun, 2022)。

「再構成せずに知能を作れるか」というのは、いま最前線で議論されているテーマそのものです。ここは後半の「正直な限界」でもう一度触れます。

### 7.3.4 (4) 行動条件付け — 自宅内製の勝敗を分ける一点

$a_t$ を遷移に注入し、舵を効かせます。ここで**行動が既知か未知かで、難易度が段違い**になります。これは自宅で世界モデルを作れるかどうかを左右する、極めて実務的な分岐なので、太字で強調します。

- **既知**(gaitlab / MuJoCo はここ。トルク指令が手元にある): $a_t$ を遷移の入力に concat するか、FiLM で変調して足すだけ。**実装が軽く、制御性も出しやすい**。
- **未知**(インターネット動画には行動ラベルが無い): 行動を教師なしで推定する**潜在行動モデル(latent action model)**が要ります。行動ラベル無しから潜在行動を抽出するこの機構は、**概念上の核心的難所**です。

ここは誇張を避けたい箇所なので正直に補足します。Genie 論文自身は latent action model を "simple and scalable" と表現しており、**11Bパラメータの大半は自己回帰のdynamics modelが占めます**——latent action は小さな別コンポーネントです。「難所」というのは「概念的に厄介」という意味で、「パラメータの大半を食う」という意味ではありません。

結論はこうです。**自宅で世界モデルを作るなら、行動ラベルが只で手に入るシミュレータ(gaitlab)から始めるのが圧倒的に有利**です(理由の桁は7.5で詰めます)。

### 7.3.5 (5) 報酬/価値ヘッド — MBRLの出口

MBRLで使うなら、潜在から報酬と価値を読む小さなヘッドを足します。

$$\hat r_t = r_\psi(z_t),\qquad \hat v_t = V_\psi(z_t)$$

Dreamer は想像ロールアウトの上で actor-critic を学びます。計画特化なら、`MuZero`(Schrittwieser et al., 2020)や `TD-MPC2`(Hansen et al., 2024)のように「報酬・価値の予測さえ合えばよく、観測再構成は要らない」という**価値等価**設計もあります。見た目を捨てて制御に全振りする、という割り切りです。

### 7.3.6 学習目的の型 — Dreamer流ELBOを読む

再構成ベースの代表として、Dreamer流の**ELBO = 変分下限(Evidence Lower Bound)**を一つだけ読んでおきます。

$$\mathcal{L} = \underbrace{-\ln p(o_t\mid z_t)}_{\text{再構成}} \;+\; \underbrace{\beta\, \mathrm{KL}\!\big(q(z_t\mid o_t,h_t)\,\|\,p(z_t\mid h_t)\big)}_{\text{予測との整合}} \;+\; \underbrace{\|\hat r_t - r_t\|^2}_{\text{報酬}}$$

第2項の**KL = カルバック・ライブラー(Kullback-Leibler)ダイバージェンス**が、この式の隠れた主役です。これは「観測を見た後で分かる潜在($q$、後知恵)」と「観測を見る前の予測($p$、事前)」を一致させる圧力になります。言い換えると、**予測できる潜在**を作らせる駆動力です。JEPA系は、この第1項(ピクセル再構成)をまるごと捨て、特徴空間の予測損失に置き換える——という設計だと読めば、系統の違いが一枚の式の上で見えてきます。

## 7.4 学習法の4系統 — 長短を正直に

5部品の組み合わせ方には、大きく4つの流派があります。ここは「どれが正解」ではなく、**目的次第で正解が変わる**ので、長所と難所を並べて正直に見比べます。

| 系統 | 代表 | 予測対象 | 長所 | 短所・難所 |
|---|---|---|---|---|
| **再構成ベース** | Dreamer系(RSSM+VAE) | 潜在+ピクセル再構成 | 少パラメータ・長期一貫性・MBRL実績豊富・**単一GPUで回る** | 背景ディテール再構成に容量浪費、鮮明さは拡散に劣る |
| **対比/潜在予測(JEPA)** | I-JEPA / V-JEPA | 特徴空間の未来/欠損 | 無関係詳細を捨て制御に効く表現、計算・ラベル効率 | 潜在崩壊回避が要る、行動制御・生成デモが作りにくい(研究途上・要検証) |
| **拡散動画予測** | GameNGen / DIAMOND | 次フレーム(条件付き拡散) | 視覚的細部・鮮明さ、神経ゲームエンジンとして体験が強い | 1フレームに複数デノイズ→**リアルタイム化と長期安定が高コスト** |
| **離散トークン自己回帰** | IRIS / Genie | VQトークン列 | 表現力・スケール則に乗る・行動未知でもlatent actionで対処 | トークナイザ品質に律速、foundation規模は**クラスタ必須** |

覚え方として、**「一貫性のDreamer、効率のJEPA、鮮明さの拡散、スケールのトークン」**と語呂で括っておくと、後で系統を思い出しやすいと思います。

### 「異常に良い結果」への注意(正直な開示)

ここは、この連載が最も大切にしている作法なので、少し丁寧に釘を刺します。論文の派手な数字を、そのまま鵜呑みにしないでください。

DIAMOND の Atari 100k 人間正規化スコア 1.46、GameNGen の **PSNR = ピーク信号対雑音比(Peak Signal-to-Noise Ratio)** 29.4(短クリップでは人間の識別が概ねチャンス並み)——どれも魅力的な数字です。しかし、**実証されているのは短〜中期ロールアウトまで**です。GameNGen 論文自身は「マルチ分スケールで安定し、人間の識別は5分後もほぼチャンス並み」と主張していますが、これの正確な読み方は「すごい」ではなく、**「実証範囲が約5分に留まる(それ以降は未検証)というスコープ限界」**です。

より長時間の自己回帰では、一般にドリフト増大が予想されます。ですがそれは、**実証済みの安定範囲**と**その先への外挿**を分けて語るべき事柄で、両者を混同してはいけません。ベンチの好結果は、必ず「どの地平・どの指標か」に還元して読む——これは第4回で8.5年の壁を疑ったのと同じ姿勢です。

## 7.5 データ — 実環境 vs シミュレータ(gaitlab が効く場所)

世界モデルの学習には、**行動条件付きの遷移データ**が大量に要ります。ここが自宅研究の、地味だけれど決定的な分岐点です。

- **実環境**: 実ロボや実映像は現実的ですが、行動ラベルが無いか、あっても高価です。だからこそ Genie は latent action という重装備を背負っています。
- **シミュレータ(MuJoCo / gaitlab)**: **行動(トルク・関節指令)が只で付いてくる**遷移を、**無限に**生成できます。しかも**外部送信ゼロのオンプレ**で完結する。読者のオンプレ主義(データを外に出さない)と、データ源の性質が完全に一致します。基礎は `MuJoCo`(Todorov et al., 2012)、実機MJCFは MuJoCo Menagerie(Go2 など)。

### フェルミ推定: シム由来データはどれだけ安いか

シミュレータでデータが「只」とは、具体的にどれくらいの桁なのか。分解して見積もります。

- **分解**: 必要遷移数を $D$、シム速度を $s$[env-step/s]とすると、収集時間 $T=D/s$。
- **仮定**: MJX(MuJoCo XLA、GPU上でバッチ並列)で中規模ロボ(~30自由度 = Degrees of Freedom)を数千env並列。ここで正直な注意を挟みます。公開MJXベンチの桁感は課題依存で**大きく振れます**——7自由度Frankaで約 $3.7\times10^4$ env-step/s、接触リッチ課題で約 $1.6\times10^5$、最速級の約 $2.7\times10^6$ は 8チップのTPU v5(=単一GPUではない)です。**30自由度・接触ありを単一コンシューマGPU(5090)で回すなら、$s$ は接触量次第で $10^4$〜$10^5$ に落ちうる**(多点接触ほど遅い。要検証・環境依存)。楽観に振れないよう、ここでは中央値 $s=5\times10^4$ を採ります。
- **概算**: $D=10^8$ 遷移なら $T\approx 10^8/(5\times10^4)\approx 2\times10^3$ 秒 = **約33分**。$D=10^9$ でも約 $2\times10^4$ 秒 ≈ **5.5時間**。
- **出所/含意**: DeepMindのMJX系報告と、一般的なGPU物理シムのオーダーから。要点は精度ではなく**桁感**です——**学習に要るデータは、5090一枚のシムなら分〜時間で採れる**。つまりボトルネックはデータ収集ではなく**モデル学習側**に移る、という健全な状態になります。

ただし正直な但し書きを。接触リッチ(把持・多点接触)はMJXの弱点で減速します。gaitlabの把持タスクでは、この見積もりを楽観しすぎないでください。歩行のような比較的接触が単純なタスクほど、この「分〜時間」の桁が効きます。

## 7.6 評価 — 何を測るべきか(4指標)

世界モデルの「良さ」は、単一の数値では絶対に出ません。ここを誤ると、綺麗な動画に騙されて数ヶ月を溶かします。最低限、次の4軸を**分けて**測ってください。

1. **ロールアウト忠実度(rollout fidelity)**: 与えた行動列に対する $k$-ステップ予測誤差。潜在系は潜在MSE、ピクセル系は PSNR / `LPIPS`(学習知覚画像パッチ類似度; Learned Perceptual Image Patch Similarity)/ `FVD`(フレシェ動画距離; Fréchet Video Distance)など。**必ず地平 $k$ を明記すること**。1ステップは全員綺麗で、勝負は $k=50, 100, \dots$ です。地平を伏せた「予測誤差」の数字は、ほぼ意味がありません。
2. **長期一貫性(long-horizon consistency)**: 自己回帰を続けたとき、物体・レイアウト・物理が保たれるか。ドリフト、消失、瞬間移動の有無。定量化は難しく、下流性能で代理することが多い軸です。
3. **行動制御性(action controllability)**: 同一初期状態から**異なる行動**を与えたとき、予測が意味的に分岐するか。行動を無視して「それらしい続き」を出すモデルは、ここで落ちます。介入(do-操作)で反実仮想を測るのが筋です。7.0の「映画 vs ゲームエンジン」を数値で判定する軸、と言い換えられます。
4. **下流タスク(downstream MBRL)**: 最終審です。世界モデル内で学んだ/計画した方策の**制御性能**と、**サンプル効率**(実環境ステップあたりの上達度)。Dreamer / IRIS / DIAMOND は Atari 100k・DMC などで、少データ制御性能を主指標にしています。gaitlab なら「想像内で学んだ歩容が、実MuJoCoで走るか」です。

**正直な原則**: 1〜3が良くても、4が伴わないことは頻繁に起きます(綺麗な動画 ≠ 制御に使える)。逆に、価値等価系(MuZero / TD-MPC2)は1・2が汚くても4が良いことがあります。だから、**目的が制御なら4を主指標に、1〜3は診断用**に回すのが健全です。指標の主従を間違えないでください。

## 7.7 内製の現実的難所3つ + フェルミ

理想の配線図を描いたら、次は現実の壁です。世界モデル内製でつまずくポイントは、経験上ほぼ3つに集約されます。うち1つ(計算)は、フェルミで「作れる/作れない」を機械的に切ります。

### 難所A: 長期ロールアウトのドリフト(誤差蓄積)

自己回帰では、各ステップの予測誤差が**次の入力になります**。1ステップの相対誤差を $\epsilon$ とすると、誤差が独立に近ければ $H$ ステップ後に $\sqrt{H}$ 程度、相関や正のフィードバックがあれば線形〜指数で膨らみます。本質は、学習分布から外れた自己生成入力にモデルが晒される**分布シフト**で、これは模倣学習の compounding error(蓄積誤差)と同型の問題です。

対策の定石はいくつかあります。学習時に自分の予測を入力に混ぜる(scheduled sampling)、潜在空間で回して誤差源のピクセル詳細を切る、離散化で誤差を吸収する(VQ / MaskGIT)、KL・正則化で予測しやすい潜在に寄せる。ただし——**「短期は綺麗、長期で崩れる」は、バグではなく仕様に近い**と受け止めてください。だからこそ評価地平を伸ばして、崩れ方を正直に出すのが誠実な報告です。

### 難所B: 行動を効かせる

モデルは楽をしたがります。放っておくと「行動を無視して尤もらしい続き」を出しがちです(特に動画事前学習が強いモデルほど顕著)。対策は、行動を各層に FiLM で強く注入する、行動を変えたペアで分岐を促す対比損失を足す、そして**行動ラベルが只のシム(gaitlab)で条件付けを先に確立してから複雑化する**——この順序が肝です。未知行動(latent action)は**最後に足す拡張**であって、出発点にしてはいけません。7.3.4の結論がここで効いてきます。

### 難所C: 計算 — フェルミで「作れる/作れない」を切る

第4回の$6ND$則をここでも道具として使います。学習総FLOPを $C\approx 6ND$($N$=パラメータ、$D$=学習トークン。出典 `Scaling Laws`(Kaplan et al., 2020)/ `Chinchilla`(Hoffmann et al., 2022))と置きます。世界モデルはRL/動画で厳密には$6ND$から外れますが、**桁の物差し**としては十分です。

5090の実効スループットは、BF16テンソル演算のdenseピークを ~209.5 TFLOPS(FP32累算相当の報告値。NVIDIAの看板公称は FP4+スパースの「3352 AI TOPS」表記で、桁の取り方が違う点に注意)とし、**MFU = モデルFLOP利用率(Model FLOPs Utilization)を 30〜50%** と置いて、実効 $\sim 6\text{–}10\times10^{13}$ FLOP/s(=60〜100 TFLOPS)。以下では保守的に $8\times10^{13}$ を使います。

#### 【作れる】小〜中規模の自作世界モデル(IRIS級)

- **分解/仮定**: IRIS の world-model Transformer の実サイズは約8Mパラメータ(10層・256次元)で、トークナイザ込みでも50Mには届きません。ここでは余裕を見て $N=5\times10^7$、$D=10^9$ トークン、10 epoch相当で $D_\text{eff}=10^{10}$ と置きます。
- **概算(FLOP物差し)**: $C=6ND_\text{eff}=6\times5\times10^7\times10^{10}=3\times10^{18}$ FLOP。時間 $=3\times10^{18}/(8\times10^{13})\approx 3.8\times10^4$ 秒 ≈ **約10時間**。

ただし、ここで正直に補正します。**実測はこの$6ND$見積もりより重くなります**。RL世界モデルの学習は、逐次imagination・トークン自己回帰・小バッチのために**MFUが低く、行列積(GEMM)FLOP律速ではない**からです。実際、同クラスのIRIS(8M)でも1ゲームの学習に単一GPUで**約3.5 GPU日**かかる、という一次報告があります。5090換算でも、**実態は1日〜数日オーダー**と見るのが誠実です。

- **含意**: それでも定性結論は変わりません。**5090一枚で作れます**(1モデルあたり数日)。DreamerV3(単一GPUで150+タスク、Minecraftのダイヤ初到達; `Mastering Diverse Domains through World Models`(Hafner et al., 2023))も単一GPU想定で、自宅規模で十分に射程内です。gaitlabの運動制御世界モデルは、まさにこのクラスに収まります。

#### 【作れない】foundation規模の生成世界モデル(Genie 11B級)

- **分解/仮定**: $N=1.1\times10^{10}$、パラメータ最適でも $D\approx 20N=2.2\times10^{11}$ トークン(Chinchilla目安)。
- **概算**: $C=6\times1.1\times10^{10}\times2.2\times10^{11}\approx1.5\times10^{22}$ FLOP。時間 $=1.5\times10^{22}/(8\times10^{13})\approx1.8\times10^{8}$ 秒 ≈ **5〜6年(単一5090)**。

データ規模についても正直に補足します。Genie は動画を**20万時間超「収集」**しましたが、実学習は filtering 後の**約3万時間**(2Dプラットフォーマ)で行っています(収集量と学習量は別物です)。この学習データから素朴に時空間トークン数を積んでも、上のChinchilla由来 $D\approx2.2\times10^{11}$ に対して同桁〜1桁程度上振れしうる範囲(トークン化の解像度・パッチ条件依存・要試算)で、いずれにせよ**結論の桁(=年オーダー)は動きません**。

- **含意**: **foundation世界モデルの一からの学習は、クラスタ案件**です。自宅では「フルスクラッチ学習」ではなく、公開重み(ライセンス要確認・商用障壁に注意)からの微調整・蒸留・小規模domain特化に舵を切るのが現実解になります。第4回の「8.5年の壁」と、まったく同じ構図です。

## 7.8 自宅5090一枚での内製ロードマップ(段階設計)

ここまでの桁感を、読者環境(RTX 5090 32GB・128GB DDR5、llcore / gaitlab / llive)へ具体的に接続します。厚くすべきは「自分の手で辿れる道筋」なので、段階で示します。

1. **段階0(着荷前・ノートCPU律速)**: gaitlab で MuJoCo ロールアウトを貯める配線を先に作ります。行動条件付き $(o_t, a_t, o_{t+1}, r_t)$ の記録形式を固め、小さなRSSMか線形注意ダイナミクス(llcoreの効率アーキ)で **1〜2ステップ予測**が学習ループとして回ることを、CPUで確認します。規模は小さく、正しさだけを検証する段階です。
2. **段階1(5090・最小の本物)**: まず、固有受容感覚(関節角・速度)ベースの**低次元世界モデル**から。画像より遥かに軽く(7.7【作れる】の下限側)、gaitlab の四足歩行で「想像内rollout → actor-critic」= Dreamer最小構成が、5090で数時間〜1日オーダーで閉じます。ここで**まず制御性能(指標4)で、世界モデルが仕事をしていることを示す**のが最優先です。
3. **段階2(画像を足す)**: MuJoCoのレンダ画像を VQ でトークン化 → IRIS流トークン自己回帰、または連続潜在 + 小さな拡散ヘッド(DIAMOND流)。ここで初めて「見て検証できる」神経シムになります。**評価地平を伸ばして、ドリフトを正直に出す**のを忘れずに。
4. **段階3(効率アーキで長く安く)**: llcore の線形注意/状態空間モデル/量子化・蒸留を潜在ダイナミクスに投入し、**長いコンテキストのrolloutをCPU完結・省メモリで**回す方向へ。ここが llcore と世界モデルの合流点であり、差別化の芯になりうる場所です。
5. **やらないこと**: Genie / Sora 級 foundation の一からの学習(7.7【作れない】)。`Sora: Video generation models as world simulators`(OpenAI, 2024)は「動画生成=世界シミュレータ」の看板ですが、規模が別物です。なお llive(自己進化記憶)は、世界モデルの**外側の記憶・メタ制御**として被せる位置づけで、モデル本体の巨大化とは別軸だと整理しておきます。

図: 内製ロードマップ4段(段階0=CPUで配線検証 → 段階1=低次元Dreamer最小 → 段階2=画像VQ/拡散 → 段階3=llcore効率アーキ)。右上に「やらない=foundation一括学習」を破線で。

## 7.9 誤解されやすい点・正直な限界

最後に、本章の主張が独り歩きしないよう、誤読しやすい点を正直に釘刺ししておきます。ここだけ読んでも価値があるように書きました。

- **「動画が生成できる=世界モデル」ではありません**。行動で分岐が変わり(制御性)、その予測が下流制御を改善して初めて世界モデルです。綺麗なrolloutに騙されない(指標4が主)。7.0の「映画 vs ゲームエンジン」に常に立ち返ってください。
- **再構成の鮮明さと、制御有用性は別物**です。ピクセルPSNR / FVDが良くてもMBRLが伸びない、逆に価値等価系(MuZero / TD-MPC2)は見た目が汚くても制御は強い。**目的から指標を選ぶ**のが鉄則です。
- **短期ロールアウトの好成績を、長期に外挿しない**。ドリフトは仕様に近い(難所A)。数字は必ず地平付きで語ってください。
- **$6ND$ は世界モデルには近似**にすぎません。RLはデータ再利用(replay)・online収集・想像rolloutでFLOP配分が違い、拡散は1ステップ複数デノイズで実効コストが上振れます。本章のGPU時間は**桁の物差し**であって、精密見積もりではありません(要ベンチ実測)。
- **RTX 5090の209.5 TFLOPS(dense BF16換算の報告値。NVIDIAの看板はFP4+スパースのAI TOPS表記)は理想ピーク**です。実学習はMFU・メモリ帯域(1792 GB/s)・32GB VRAM上限に律速され、実効は3〜5割。さらに逐次RL世界モデルはGEMM FLOP律速でないため、$6ND$換算のGPU時間は楽観側に外れやすい(難所C)。大バッチの動画拡散は、VRAMですぐ頭打ちになります(要検証・実測前提)。
- **公開世界モデル重みの商用利用は、ライセンス要確認**です。研究再現と商用製品化で条件が変わります(読者の商用障壁観点)。フルスクラッチが無理な規模では、ライセンスが緩い基盤や自作蒸留で回避する設計判断が要ります。
- **JEPA系の「行動条件付き生成世界モデル」としての成熟度は、発展途上(要検証)**です。表現学習としては強力ですが、制御可能な生成デモは再構成/拡散系ほど確立していません——採用は目的に照らして判断してください。

## 7.10 まとめと、次回予告

今回の持ち帰りを、3行に畳みます。

- **世界モデルは5部品の配線図**: 観測エンコーダ・潜在ダイナミクス・デコーダ/潜在予測・**行動条件付け**・報酬価値ヘッド。内製とは各ブロックを何で埋めるかを決めること。
- **自宅で作れる/作れない**: 制御向けの小〜中規模世界モデル(Dreamer / IRIS級)は**5090一枚・数日で作れる**。foundation級の汎用映像世界モデル(Genie 11B級)は**5〜6年で桁が違い、作れない**。境界は行動ラベルの有無とパラメータ規模で決まる。
- **最短経路はシムから**: 行動が只で付くgaitlab(MuJoCo)で条件付けを先に確立し、低次元→画像→効率アーキの順に段階を上る。評価は「地平付きの4指標」、主役は下流制御(指標4)。

この連載はずっと「自宅から世界モデルへ」と副題を掲げてきました。第4回で器の大きさを測り、蒸留で知識を移し替え、そして今回、ついに**その器の中でエンジンを組む配線図**まで辿り着きました。骨格はもう、あなたの手の中にあります。

——ですが、まだ一番おいしいところが残っています。世界モデルは「作る」だけでは半分です。本当の醍醐味は、**その頭の中のシミュレータの中で、エージェントが夢を見ながら賢くなる**瞬間にあります。実世界で一度も転ばずに、想像の中だけで歩き方を覚える。ロボットが「まだ起きていない失敗」から学ぶ。その錬金術めいたループ——Dreamerが「imagination(想像)」と呼んだ工程——は、本当にサンプル効率を数十倍にするのか。そして、どこで嘘をつくのか。
