# 第8回 生成AIの部品が世界モデルになる — 橋渡し

> 連載「**生成AIを内製する 〜自宅から世界モデルへ〜**」の第8回です。

## この回で得られること(導入)

ここまでの連載で、私たちは生成AIを一度バラバラに分解してきました。Transformer と自己注意、位置符号、自己回帰復号、拡散・flow、事前学習、RLHF/GRPO、そして推論最適化。第1回から第7回まで、まるで時計を裏返して歯車を一つずつ取り出すように、「部品」として並べてきたわけです。

そして本連載のサブタイトルには「〜自宅から世界モデルへ〜」と書きました。ここまで読んでくださった方の中には、こう感じた方がいるかもしれません。「世界モデルって、いよいよ何か新しい巨大なアルゴリズムが出てくるのだろう」と。

本回の主張は、実はその逆です。一言で言えば、こうなります。

> **これらの部品は、既に世界モデル(world model)を内製するための部品箱である。**

世界モデルは、新しい万能アルゴリズムではありません。既に私たちが理解した部品を、「行動で条件づけられる、時間発展する生成器」として組み替えたものにすぎません。したがって、ここまで積み上げてきた生成AIの内製ノウハウは、そのまま世界モデル内製の土台になります。

この回を読むと得られるものは、次の3つです。

1. 前半の各章(自己注意・トークン化・拡散・RLHF…)が、世界モデルの**どの箱に再利用されるのか**という写像の全体図。
2. その写像を、手元の実プロジェクト — gaitlab(物理AIの歩行進化)、llcore(効率アーキ)、llive(自己進化記憶) — に**具体的に接続**する道筋。
3. そして最も大事な、「**自宅規模で作れるもの/作れないもの**」を誇張せず切り分ける桁感覚(フェルミ推定つき)。

つまりこの回は、連載の前半(生成AIの解体)と後半(世界モデルへの再構築)をつなぐ「橋」です。橋の向こう側がどんな地形なのかを、まずは俯瞰します。

## 用語ミニ辞書

先に、この回で何度も出てくる言葉を「日本語(English)」表記で並べておきます。ここを一度通しておくと、後半の密度の高い議論が驚くほど軽く読めるようになります。

- **世界モデル(world model)**: 環境の状態と、行動 $a_t$ を入力に、次の状態・次の観測・報酬を予測する学習済みのダイナミクス(時間発展の法則)。式で書けば $p_\theta(o_{t+1}, r_{t+1} \mid o_{\le t}, a_{\le t})$ を近似する生成モデルです。平たく言えば「頭の中のシミュレータ」。
- **潜在ダイナミクス(latent dynamics)**: 生の観測(画素やセンサ値)を潜在ベクトル $z_t$ に圧縮し、その潜在空間の中で時間発展 $z_t \to z_{t+1}$ を予測する方式。後述の Dreamer 系の中核です。
- **モデルベース強化学習(model-based RL)**: 環境と直接やり取りしながら方策を学ぶ「model-free」に対し、**学習した世界モデルの中で**方策を鍛える手法。
- **想像上のロールアウト(imagined rollout / dreaming)**: 実環境を1ステップも動かさず、世界モデルの中だけで軌跡 $z_t, a_t, z_{t+1}, \dots$ を展開すること。文字どおり「夢の中で練習する」。
- **潜在行動モデル(latent action model, LAM)**: 行動ラベルの無い動画から、連続するフレーム間で「何が起きたか」を教師なしで**離散的な行動コード**として推定する仕組み。後述の Genie の鍵です。
- **時空間Transformer(spatiotemporal transformer)**: 空間(フレーム内のパッチ/トークン)と時間(フレームの系列)の両方の軸に注意を張る Transformer。計算量を減らすため、空間注意と時間注意に**分解(factorized attention)**することが多いです。
- **ニューラルゲームエンジン(neural game engine)**: 描画エンジンをニューラルネットで置き換え、行動入力に応じて次フレームを生成する系。後述の GameNGen が代表例。
- **テスト時計算による計画(planning as test-time compute)**: 推論時に、世界モデルの中で先読み探索(木探索やモデル予測制御)を回し、計算を余分に「消費」して良い行動を選ぶこと。o1 型の「思考時スケーリング」を、言語ではなく「想像の中の未来」に対して行う版だと思ってください。
- **MFU(Model FLOPs Utilization)**: 理論ピーク FLOPS に対する実効利用率。単GPU・小モデル・小バッチだと 20〜40% 程度まで落ちることが多い指標です(後のフェルミ推定で使います)。
- **KV cache(Key-Value キャッシュ)**: Transformer が過去のトークンの Key/Value を保持しておく作業メモリ。文脈が長くなるほど線形に膨らむ、後半の主役です。

## かみくだき — 「生成AIは、もう世界モデルの部品箱になっている」

言葉で直感を作りましょう。

言語モデル(LLM)は「**次のトークン**」を予測します。世界モデルは「**次のフレーム/次の状態**」を、しかも**あなたのボタン入力(行動 $a_t$)に応じて**予測します。両者の骨格は、実はまったく同じ自己回帰(または拡散)です。違いはたった2つ。

- **条件づけに行動 $a_t$ が入る**こと。
- **トークンが「言葉」ではなく「世界の断片」**であること。

料理の比喩で言えば、こうなります。LLM は「文章という1次元の世界」の物理法則(=言語の統計)を学んだ生成器です。世界モデルは「画素とセンサという高次元の世界」の物理法則を学んだ生成器です。**同じ料理法**(自己注意で文脈を混ぜ、自己回帰で次を吐く)で、**素材**(トークンの中身)と**味付け**(行動での条件づけ)を替えただけ、と捉えて構いません。

> 図の意図: 「LLM=文章の物理を学んだ生成器」「世界モデル=画素とセンサの物理を学んだ生成器」を左右に並べ、同じ骨格(自己注意+自己回帰)を共有し、入力に行動 $a_t$ が差し込まれる一点だけが違う、という対比図。

### 原点にある最小構成 — World Models (Ha & Schmidhuber, 2018)

この直感を、最も小さな構成で示したのが世界モデルの原点、**World Models (Ha & Schmidhuber, 2018)** です。構成はこうです。

1. 観測を **VAE** で潜在に圧縮する(エンコーダ)。
2. **RNN(MDN-RNN)** で潜在の時間発展を予測する(ダイナミクス)。
3. その上で、小さなコントローラを進化戦略(**CMA-ES**)で鍛える(方策)。

ここで、**2つの実験を混同しないこと**がとても大事です。よく「夢だけで学習して実環境に転移した」と一括りに語られますが、一次文献を丁寧に読むと、成立の仕方が環境ごとに違います。

- **VizDoom(DoomTakeCover)**: 一度も実環境を触らず、「夢」の中だけで学んだ方策が、そのまま実環境で動きました(夢のみ学習からの実環境転移。報告値は実環境スコア **1092±556**)。これが「夢で練習して本番で通用した」の成立側です。
- **CarRacing**: こちらはワールドモデル(VAE+MDN-RNN)を**圧縮表現=特徴抽出器**として使い、方策(コントローラ)は**実 CarRacing 環境で** CMA-ES 学習しました(実環境スコア **〜906**)。

つまり「夢だけで学習して実環境転移」を両方の環境に帰属させるのは、一次文献の誤読です。成立したのは VizDoom 側だ、と正確に言い分けておきます(こういう細部を潰しておくのが、この連載の作法です)。

いずれにせよ、ここで重要なのは、**世界モデル内製の全要素が既に揃っている**という事実です。エンコーダ(トークナイザ)、ダイナミクス(自己回帰器)、その上の方策最適化(RL)。前半で学んだ部品は、まさにこの3つの箱を高性能化する方法そのものだったのです。

```
   観測 o_t ──[トークナイザ/VAE]──▶ 潜在 z_t
                                      │
                行動 a_t ────────────▶├─[ダイナミクス:Transformer/SSM/拡散]
                                      ▼
                                    z_{t+1}, r_{t+1}  ← ここを何度も回すと「想像」
                                      │
                                      └─[デコーダ]──▶ 予測観測 ô_{t+1}
              方策 π(a|z) を「想像上のロールアウト」で最適化(RL/GRPO/計画)
```

上の図で「$z_{t+1}$ を出す → それをまた入力に戻す → また次を出す」というループを、実環境に一切触れずに回すこと。これが「想像(dreaming)」です。この単純なループの中に、前半で学んだ部品が総動員される、というのがこの回の背骨になります。

## 詳細・原理 — 部品対応表・世界モデルの3系統・フェルミ推定

### (A) 部品対応表:前半の各章が、世界モデルのどこに再利用されるか

まず、この回の心臓部です。前半で分解した部品が、世界モデルのどの役割に再利用されるのかを一枚の表にします。

| 前半で学んだ部品 | 世界モデルでの役割 | 代表例・典拠 |
|---|---|---|
| 自己注意 / Transformer | フレーム・トークン履歴に対する時空間注意の骨格 | IRIS (Micheli et al., 2023), Genie (Bruce et al., 2024) |
| 位置符号(RoPE 等) | 空間位置(パッチ座標)+ 時間位置(フレーム番号)の符号化。時空間で分解 | Sora (OpenAI, 2024) の spacetime patches |
| 自己回帰復号 | 「次フレーム/次潜在トークン」をボタン入力で条件づけて逐次生成 | Genie, IRIS, GameNGen の frame conditioning |
| 拡散・flow | 次フレームそのものの高忠実生成(観測モデル) | GameNGen (Valevski et al., 2024), DIAMOND (Alonso et al., 2024) |
| トークン化(VQ-VAE/VQ-GAN) | 観測を離散潜在コードに圧縮=世界モデルの「表現モデル」 | VQ-VAE (van den Oord et al., 2017), VQ-GAN (Esser et al., 2021) |
| 事前学習(自己教師あり) | 行動ラベルなしの大規模動画から自己教師あり表現を学習し、ダイナミクス学習の土台にする(行動条件づけ dynamics は次段で足す) | V-JEPA (Bardes et al., 2024, 潜在表現/エンコーダ), V-JEPA 2 (Meta AI, 2025, 行動条件づけ world-model/計画), Genie の LAM |
| RLHF / GRPO(方策最適化) | 学習済みモデルの「夢の中」で actor-critic / 選好最適化 | DreamerV3 (Hafner et al., arXiv 2023; Nature 2025), GRPO (Shao et al., 2024) |
| 推論最適化(KV cache/量子化/蒸留) | 実時間ロールアウトと長期記憶のメモリ律速を緩和 | ← llcore の主戦場(後述) |

この表が、本回の背骨です。強調しておきたいのは次の一点。

> **世界モデルを一から発明する必要はありません。既に持っている部品の「条件づけ」と「素材」を替えるだけで、骨格ができあがります。**

### (B) 世界モデルの3系統(+計画)

いまの世界モデルは、「ダイナミクス(時間発展の部分)に、前半のどの生成方式を採るか」で大きく3系統に分けられます。ここは丁寧に見ていきます。

#### 系統1: 潜在ダイナミクス + 想像(Dreamer 系, model-based RL)

観測を潜在 $z_t$ に圧縮し、潜在空間で再帰的に時間発展させ、その中で actor-critic を回す系統です。代表格の **DreamerV3 (Hafner et al., arXiv 2023; Nature 2025)** は、**単一のハイパーパラメータ設定**で150以上のタスクを解き、**人間のデータなしに Minecraft のダイヤモンド採掘を達成**しました。

ここも典拠を正確に。preprint は arXiv 2023 の "Mastering Diverse **Domains** through World Models"、査読版は別タイトルの Nature 2025 "Mastering diverse **control tasks** through world models" です。年と掲載誌、そしてタイトルを混同しないでください(細部の正確さが信頼の通貨です)。

潜在空間で回すため計算とメモリが軽く、**単GPUで訓練可能**。これは後で「自宅で作れる」側に効いてきます。そして gaitlab のような、状態(関節角・接触)ベースの制御に最も素直に載る系統でもあります。

#### 系統2: 自己回帰トークン世界モデル(Genie / IRIS)

フレームを離散トークン列に変換し、Transformer で次トークンを自己回帰生成する系統です。**IRIS (Micheli et al., 2023)** は Atari 100k(実環境わずか10万ステップ、≒ 人間2時間相当)で高いサンプル効率を示しました。

そして **Genie (Bruce et al., 2024)** が画期的だったのは、ここに**潜在行動モデル(LAM)**を足したこと。**行動ラベルの無いインターネット動画から、「操作可能な2D世界」を教師なしで学んだ**のです。誰もボタン操作を教えていないのに、動画のフレーム間変化から「これは右移動、これはジャンプ」に相当する離散コードを勝手に発見する — これが LAM の魔法です。

系譜は速く進みます。**Genie 3 (Google DeepMind, 2025)** は、**720p・24fps・数分の一貫性**を持つ実時間ナビゲート可能な環境を、テキストから生成すると報告されています。初代 Genie は低フレームレート(=非実時間)、Genie 2 は10〜20秒の持続でしたから、実時間性と持続時間が急伸したことになります(※初代の具体的な生成 fps は一次情報での確認が限定的=**要検証**)。ただし、これはクラスタ級の産物であって、**自宅規模の到達点ではありません**(この線引きは後で詳述します)。

#### 系統3: 拡散/flow による動画予測(GameNGen / DIAMOND / Sora / Cosmos)

次フレームを拡散モデルで生成する系統です。

- **GameNGen (Valevski et al., 2024, "Diffusion Models Are Real-Time Game Engines")** は Stable Diffusion を土台に **DOOM を 20fps** でニューラル生成し、人間の識別者が「実ゲームか生成か」を見分ける実験で、**ほぼ偶然当てるレベル**(5分の自己回帰生成後でも)まで到達しました。描画エンジンをニューラルネットが丸ごと肩代わりした、という象徴的な結果です。
- **DIAMOND (Alonso et al., 2024, NeurIPS Spotlight)** は、「離散潜在への圧縮が、RL に効く**視覚的細部**を捨ててしまう」という問題を、拡散で回避しました。結果、Atari 100k で人間正規化スコア(HNS)平均 **1.46** を出し、STORM 1.266 / DreamerV3 1.097 / IRIS 1.046 を上回りました。しかも **13M パラメータ**という軽量さで、です。
- **Sora (OpenAI, 2024)** は自らを "world simulators" と位置づけ、**Cosmos (NVIDIA, 2025)** は物理AI向けの「世界基盤モデル(world foundation model)」を掲げています。

系統3は視覚忠実度が高い一方で、拡散の反復サンプリングゆえに**実時間ロールアウトの計算が重い**という宿命があります。ここも後半の伏線です。

#### 計画=テスト時計算による想像(MuZero / TD-MPC)

3系統とは少し軸が違う、「学習した世界モデルの**中で先読み探索する**」系統も押さえておきます。

- **MuZero (Schrittwieser et al., 2020)** は、学習済みモデルの上で MCTS(モンテカルロ木探索)を回し、囲碁・将棋・Atari を制しました。
- **TD-MPC2 (Hansen et al., 2023)** は、潜在空間でモデル予測制御(MPC)を行い、連続制御・移動タスクで高いサンプル効率を示します。

これは「推論時に計算を余分に消費して良い行動を選ぶ」という構図です。つまり、LLM の思考時スケーリング(chain-of-thought / o1)を、**言語ではなく「想像上の未来」に対して行う**版だと理解できます。そしてこれこそ、「GRPO 等の RL が『世界モデル上の方策最適化』に効く」という写像の核心でもあります。前半でやった GRPO は、捨てずにここへ持ってこられるわけです。

> 図の意図: 世界モデル3系統(潜在ダイナミクス / 自己回帰トークン / 拡散)+計画 を、横軸「ダイナミクスの生成方式」・縦軸「計算コスト vs 視覚忠実度」で配置し、それぞれの代表モデルをプロットした地図。

### (C) フェルミ推定その1 — 「自宅で小さな世界モデルを訓練する」コスト

ここからは桁感覚の話です。抽象論を「時間」と「GB」に落として、地に足をつけます。

**問い**: RTX 5090 で、$N=10^8$ パラメータの世界モデルを $D=10^{10}$ トークン(フレーム × フレームあたりトークン)訓練したら、何時間かかるでしょうか。

**分解**: 訓練 FLOPs は経験則 $C \approx 6ND$ で概算できます(順伝播+逆伝播で、トークン当たり約 $6N$ FLOPs)。この近似式の初出は **Scaling Laws for Neural Language Models(Kaplan et al., 2020)** で、**Chinchilla(Hoffmann et al., 2022)** もこれを踏襲します(ただし Chinchilla の主眼は式そのものではなく、$N/D$ の compute-optimal な配分の方です)。

$$
C = 6 \times 10^8 \times 10^{10} = 6\times10^{18}\ \text{FLOPs}
$$

**仮定・出所**: RTX 5090 のピーク dense BF16 tensor スループットは、実測系の報告で **約 210 TFLOPS**($=2.1\times10^{14}$ FLOP/s)とします。ここは注意が要ります。GeForce Blackwell の tensor コアは **FP32 累算時に半速**になるため、学習相当の dense BF16(FP32 accumulate)ではこの ~210 になります。FP16 累算やスパースなら ~419 も出ますが、公称の大きい値との混同に注意してください(半速の主因は FP32 累算)。さらに、単GPU・小モデルの実効率 MFU を **30%** と仮定します。すると実効スループットは:

$$
2.1\times10^{14} \times 0.30 \approx 6.3\times10^{13}\ \text{FLOP/s}
$$

**概算**:

$$
\frac{6\times10^{18}}{6.3\times10^{13}} \approx 9.5\times10^{4}\ \text{秒} \approx \textbf{約26時間(桁として ~1日)}
$$

**結論(桁感)**: **1億パラメータ級・100億トークン級の世界モデルは、5090 一枚で1〜数日**で訓練できる射程にあります。DreamerV3・DIAMOND・TD-MPC2 が「単GPUで論文再現可能」と言われるのは、まさにこの帯域にあるからです。これは、自宅で現実に手を動かせるスケールだ、と胸を張って言えます。

> 図の意図: $C \approx 6ND$ を三角形(N × D × 6)で図示し、5090 の実効 FLOP/s で割ると「約1日」に着地する、という桁の階段を可視化。

### (D) フェルミ推定その2 — 長ロールアウトのメモリ律速(llcore への伏線)

さて、訓練コストは1日で済みました。では、訓練した世界モデルを実際に「長く想像させる」と何が起きるでしょうか。ここに、次節 llcore への伏線があります。

**問い**: 自己回帰トークン世界モデルで「**1分間の想像**」を保持するとき、Transformer の KV cache は何 GB になるでしょうか。

**分解**: 1分 × 24fps = 1440 フレーム。1フレームを VQ で $16\times16=256$ トークンに符号化すると、文脈長は:

$$
L = 1440\times256 \approx 3.7\times10^{5}\ \text{トークン}
$$

KV cache のサイズは、$2\,(K,V)\times n_\text{layers}\times L\times d_\text{model}\times \text{bytes}$ です。

**仮定**: $n_\text{layers}=24,\ d_\text{model}=1024,\ \text{BF16}=2$ バイト。

**概算**:

$$
2\times24\times(3.7\times10^{5})\times1024\times2 \approx 3.6\times10^{10}\ \text{バイト} \approx \textbf{約34GB}
$$

**結論**: **たった1分の想像を、素の全注意(full attention)で保つだけで、5090 の 32GB を超えてしまいます**。GQA(grouped-query attention)を使えば数倍は削れますが、文脈長 $L$ に**線形に増える**という性質そのものは消えません。

この「$L$ に線形」という重石こそ、次節 08.4 の主題です。線形注意 / 状態空間モデルを使えば、文脈長に依らず**定数サイズの再帰状態**で済み、長ロールアウトのメモリ律速が**原理的に外れる**。訓練は1日でできても、長く想像させ続けるにはメモリの壁がある — この壁を llcore が壊しにいく、という筋書きです。

### (E) gaitlab への具体接続 — 進化/RL 歩行最適化に潜在ダイナミクスを載せると何が変わるか

ここからは、抽象論を手元プロジェクトに接続します。まず gaitlab(物理AIの歩行進化)です。

gaitlab は現在、MuJoCo(Menagerie の Go2 等、実機 MJCF)上で歩行・把持を**進化 + RL** で直接最適化しています。ここに Dreamer 型 / TD-MPC2 型の学習ダイナミクスを載せると、原理的には次が変わります。

1. **想像上のロールアウトによるサンプル効率**: 方策更新の一部を「夢の中」で行い、高コストな実 sim ステップを節約する。
2. **潜在空間での計画**: TD-MPC2 のように潜在で MPC / 木探索を回し、テスト時計算で歩容(歩き方)を先読み最適化する。

**ただし、正直な限界を隠さず書きます(ここは誇張してはいけない箇所です)。**

Dreamer 系の教科書的な勝ち筋は「実環境ステップが**高価**な場合のサンプル効率」です。ところが、**MuJoCo は既に高速・安価・高精度**なシミュレータです。さらに 5090 が着荷すれば、**MuJoCo MJX(GPU)で数千環境を並列**でき、これは「近似バイアスのない厳密な」サンプルを大量に供給する手段になります。学習した世界モデルは所詮ダイナミクスの**近似**ですから、正確な本物のシミュレータが安く大量に回せる環境では、わざわざ近似を挟む意味が薄れるのです。

したがって、gaitlab で世界モデルの価値が出るのは、**生のスループットではなく**、次の3点に絞られます。

- (a) 潜在での計画/探索(テスト時計算で先読みする)。
- (b) sim2real(実機は高価で壊れやすい。**そこでこそ imagined rollout が効く**)。
- (c) 接触ダイナミクスがボトルネックのタスク。

技術選定の含意も明確です。**gaitlab は、画素ベースの Genie/GameNGen より、状態(固有受容感覚)ベースの Dreamer/TD-MPC2 系が素直に載ります**。関節角や接触力といった状態量を直接扱えるなら、わざわざ画素を経由する必要はありません。

honest な第一歩としては、まず **TD-MPC2 を1タスクでベースライン化**し、「MJX 大量並列の model-free」と「学習ダイナミクスの計画」を**同一予算で公平比較**すること。もし異常に良い結果が出たら、勝った気になる前に、model のバイアスがベンチの抜け道を漏らしていないか内訳を疑う — この作法は連載を通じて繰り返しているとおりです。

### (F) llive への接続

もう一つの手元プロジェクト、llive(自己進化型記憶フレームワーク)から見ると、世界モデルは「**予測する記憶基盤**」に相当します。

エージェントの経験を潜在軌跡として蓄え、次に起こることを予測し、想像の中で方策を試す。この構図で、**llive の4層メモリのうち「予測・シミュレーション層」を世界モデルが担う**、と位置づけられます。逆方向の相補もあります。**llive の長期記憶が、世界モデルの一貫性(数分を超える持続。これは Genie 系の弱点でした)を補う**のです。数分で崩れる想像を、外部記憶がアンカーとして支える、というイメージです。

実装上の接点は明確で、「エピソード記憶からの潜在再生 → imagined rollout での再学習」というループになります。記憶と想像が互いを補強し合う、この設計が llive と世界モデルの結節点です。

## llcore への接続 — 効率アーキが、長ロールアウトのメモリ律速を緩める

ここが、本回の主張「生成AIの内製ノウハウ = 世界モデル内製の土台」が、最も具体的に現れる場所です。

(D)で見たとおり、自己回帰世界モデルの実用律速は、**文脈長に線形増加する KV cache**(1分で ~34GB)でした。llcore が狙う効率アーキ群は、まさにこの一点に効きます。

- **線形注意 / 状態空間モデル(SSM, Mamba 系)**: 全注意の $O(L^2)$ 計算・$O(L)$ メモリを、**定数サイズの再帰状態**に置き換えます。世界モデルのロールアウトは本質的に逐次(1フレームずつ)ですから、推論には SSM の再帰形が理想的にはまります。メモリは 34GB から、状態サイズ($d_\text{state}\times d_\text{model}\times n_\text{layers}$ 程度)の**数百MB級**へ、しかも $L$ に**非依存**で落ちます。ここが効くほど、つまり**長く想像するほど、SSM の相対優位が拡大する**わけです。
- **蒸留(distillation)**: 大きな拡散/自己回帰世界モデルを、実時間ロールアウト可能な軽量学生に蒸留します。拡散なら少ステップ化(consistency / 蒸留サンプラ)がこれに当たります。GameNGen が 20fps を出せた背景にも、この生成ステップ削減があります。
- **量子化(quantization)**: 潜在ダイナミクスを INT8 / FP8 / FP4(5090 は FP4 ネイティブ)で回し、オンプレ実時間ロールアウトのメモリ/帯域を圧縮します。

**フェルミ(帯域感)**: 実時間 24fps を出すには、1フレーム当たり **~42ms** の予算です($1000\text{ms}/24 \approx 41.7\text{ms}$)。全注意では $L$ が伸びるほど1フレームの注意コストが増え、実時間性が崩れます。ところが SSM は1フレームのコストが $L$ に依らず一定なので、**任意長の想像を一定 fps で回せます**。

つまり llcore の「線形注意 / SSM / 蒸留 / 量子化で、CPU 完結・省メモリ」という設計思想は、そのまま**長ロールアウト世界モデルを自宅ハードで走らせるための前提技術**になります。llcore は単なる「軽いLLM」を作っているのではなく、**自宅で世界モデルを長時間夢見させるための土台**を作っていた、という読み替えができるのです。

## 正直な限界(honest disclosure)

この連載の核は「異常に良い結果が出たら、勝った気になる前に必ず内訳を疑う」です。世界モデルは夢のある話題だからこそ、ここは冷静に線を引きます。

- **「世界モデル = 動画生成」ではありません。** Sora 級の美麗な動画であっても、**行動で条件づけられ、その帰結が一貫する**という制御性が無ければ、それは world "model" ではなく world "**video**" です。世界モデルの難所は画質ではなく、**行動条件づけ・長期一貫性・物理的整合性**にあります。Genie 3 の「数分の一貫性」という報告も、裏を返せば「**数分を超えると崩れる**」という現状の限界を示しています(要検証: 公表は動画やブログが主体で、査読済みの定量詳細は限定的です)。
- **HNS などのベンチ数値を、一般世界モデルへ外挿しないこと。** DIAMOND の HNS 1.46 は、Atari 100k という**狭い視覚ゲーム集合**での値であって、「一般的な世界を予測できる」ことを意味しません。honest disclosure の原則どおり、異常に良いスコアは「そのベンチ特有の抜け道を学んでいないか」を疑うべきです。
- **自宅で作れるもの/作れないものの切り分け(フェルミ)。**
  - **作れる**: DreamerV3 / DIAMOND / TD-MPC2 級(単GPU・1〜数日、(C)参照)。gaitlab 用の状態ベース潜在ダイナミクス。2D の玩具 Genie 風。
  - **作れない**: Genie 3 / Sora / Cosmos 級の汎用動画世界モデル。
  - **フェルミ(規模の壁)**: 5090 の dense BF16 ~210 TFLOPS(FP32 累算)に対し、H100(SXM)は dense BF16 ~990 TFLOPS。つまり **5090 ≒ H100 の約1/5**(桁として。出所は各社公称の dense BF16 tensor 値で、GeForce 側は FP32 累算半速を反映)。フロンティア動画世界モデルの訓練を仮に「$10^4$ H100-GPU-日」規模と置くと(要検証: 公表値が乏しく桁の仮置き)、5090 一枚では $\approx 5\times10^4$ 枚日 ≈ **100年超**。仮に $10^3$ H100-日でも十数年です。→ 結論は honest に、「**借りる(クラウド spot / MJX-GPU)か、小さく作るかの二択**」。「フルスクラッチの汎用世界モデルを自宅で」は不可、「**特定タスクの実用世界モデルを自宅で**」は可、というのが正確な線引きです。
- **データとライセンスの現実。** 世界モデルは、行動付きの軌跡か、行動なしの大規模動画+潜在行動モデルを要します。前者は自前で MuJoCo から安価に生成できます(これが gaitlab の強みです)。後者の「インターネット動画で事前学習」は、権利・帯域・キュレーションのコストが大きい。既存の世界基盤モデル(Cosmos 等)を土台にする場合は、**重みのライセンス(商用可否)を必ず確認**してください。Qwen 依存が商用障壁になったのと**同型のリスク**が、世界モデルの配布重みにも存在します。オンプレ主義(データを外に出さない)を貫くなら、MuJoCo 由来の自前軌跡で状態ベース世界モデルを作る経路が、最もクリーンです。
- **Dreamer を過大評価しないこと(再掲・重要)。** 「学習ダイナミクスを載せれば必ずサンプル効率が上がる」は誤りです。安価で正確なシミュレータ(MuJoCo / MJX)が既にある環境では、**model-free + 大量並列が世界モデルを上回ることが普通にあります**。世界モデルの本当の配当は、**実機コスト・計画・転移**に現れます。導入の判断は、必ず「同一予算での公平比較」を経てから下してください。

## 本節のまとめ — 内製ノウハウから世界モデルへの一本線

長い回になりましたので、一本の線に束ねます。前半で分解した部品は、次の対応で世界モデルに直結します。

- 自己注意 = 時空間ダイナミクスの骨格
- トークン化 = 観測エンコーダ
- 自己回帰/拡散 = 次状態の生成
- 事前学習 = 無ラベル軌跡からのダイナミクス学習
- RLHF/GRPO = 想像の中の方策最適化
- 推論最適化 = 実時間・長期記憶

世界モデルとは、要するに「**行動で条件づけた生成器**」です。だからこそ、**あなたが生成AIを内製できるなら、世界モデルの骨格も内製できます**。

手元プロジェクトへの落とし方を、桁感つきで再掲します。

- **llcore**: 線形注意/SSM/蒸留/量子化で、「長ロールアウトのメモリ律速(1分の想像で ~34GB → $L$ 非依存の定数状態)」を外す前提技術を供給する。
- **gaitlab**: 状態ベースの Dreamer/TD-MPC2 を1タスクで導入し、MJX 大量並列の model-free と**同一予算で honest 比較**する(価値は throughput ではなく、計画・sim2real にある)。
- **llive**: 世界モデルの長期一貫性を記憶で補い、予測層として統合する。

最終目標の「世界モデル内製」は、フロンティア級を自宅で作ることではありません。**特定タスクで実用に足る世界モデルを、既に持っている生成AI部品から組み上げること** — その第一歩は、5090 一枚で1〜数日、今日から踏み出せる射程にあります。
