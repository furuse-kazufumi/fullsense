# 投稿 cheat-sheet — QIITA_18 / QIITA_19 を Qiita Web UI に投稿する手順

> 本ファイルは投稿時のミスを減らすための one-shot ガイド.
> 投稿完了後に削除しても OK (Git 履歴に残る).

---

## 1. 共通: Qiita Web UI 投稿手順

1. <https://qiita.com/drafts/new> を開く
2. **タイトル**: 下記の各記事 §2/§3 のタイトルをコピペ
3. **本文**: 該当 `.md` ファイルを開き、**「2 行目から指定行の手前まで」をコピペ**
   (タイトル行と「## 投稿時の推奨タグ」以降は含めない)
4. **タグ**: 5 個を 1 つずつ入力
5. **公開設定**:
   - 最初は **「限定共有投稿」** で投稿してプレビューを確認
   - 問題なければ **「全体公開」** に切替
6. **ライセンス**: 「CC BY」 (Qiita default)
7. **コメント許可**: ON

## 4. 投稿順序と間隔の推奨

選択肢 (お好みで):

| 戦略 | 説明 |
|---|---|
| **連続投稿 (即時)** | 18 → 19 を同日 30 分以内に. SNS 拡散時に「2 部作」訴求でき、相互流入も狙える |
| **1 日空け** | 19 (一般版) を先に出して反応を見て、18 (技術版) を翌日 | 「先に技術詳細を出してから一般向け」も可 |
| **時間帯を分ける** | 同日 朝に 19、夜に 18. 異なる読者層に届けやすい |

「ストーリーの面白さが全て」(memory `feedback_qiita_long_form`) を踏まえると、
**19 (一般版) を先に出す**のが共感入口として強い気がします. ご判断にお任せ.

## 5. 投稿後の最低限フォロー (5 分作業)

1. 自分の記事 URL を確認、リンク不備が無いか
2. Mermaid 図がちゃんとレンダリングされているか確認 (Qiita はネイティブ対応)
3. タグが 5 個入っているか
4. (任意) LinkedIn / X / Bluesky に Qiita URL を 1 投稿でシェア
   - memory `feedback_linkedin_translation_jp_only` で LinkedIn は日本語のみで OK
5. 公開後 24-48 時間は LGTM / コメントが付く可能性が高いので、コメントには丁寧に返信

## 6. 投稿時に避けるべきこと (sanity check)

- [ ] 本文に **D ドライブパス (D:\\... or D:/...)** が含まれていない
      ([[feedback_no_local_path_in_public]])
- [ ] **画像 placeholder (`📷 [画像 placeholder: ...]`)** が残っていない
      ([[feedback_no_image_placeholders]])
- [ ] **個人特定情報 (フォロワーの username 等)** が含まれていない
- [ ] **改訂履歴** が本文に含まれていない (本文末尾は §おまけ 等で締める)
- [ ] **推奨タグセクション** が本文に含まれていない (Qiita Web UI で別途入力)

両記事とも 2026-05-18 時点で上記すべて OK 確認済 (commit ee9a4a5 以降).

## 7. 投稿失敗 (rollback) パターン

万一投稿後にミスに気づいた場合:

- 「限定共有」のまま気づけば: 編集して修正 → 公開
- 全体公開後にミス: 編集して修正、または一旦「限定共有」に戻す
- 拡散済の重大ミス: 訂正記事を別途上げる、元記事末尾に「※ 訂正」追記

memory `feedback_benchmark_honest_disclosure` の通り、間違いを隠さず明示
するのが信用を保つ.
