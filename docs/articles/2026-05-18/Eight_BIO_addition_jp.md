# Eight 名刺プロフィール — 自己紹介文への追記案

**配置場所**: Eight 名刺アプリの「自己紹介」欄
**前置き**: AI 関連は **趣味領域** であることを明示 (本業は精密計測 / 画像処理 / 三次元計測)
**長さ**: 短中長の 3 案 (Eight UI に応じて選択)

---

## 案 A — 短い 1 行版 (~50 字)

```
趣味で OSS の LLM フレームワーク llive を 1 人開発中。Apache 2.0。
```

→ 名刺の最後にさらっと 1 行入れるイメージ。説明しすぎず、関心ある人だけ拾える深度。

---

## 案 B — 中くらい 2-3 行版 (~120 字)

```
余暇で OSS の LLM フレームワーク llive (FullSense umbrella の中核) を 1 人開発しています。
本業 (精密計測 / 三次元計測) で培った 30 年の経験を「第二の脳」として AI 開発に活かす方針。
Repo: https://github.com/furuse-kazufumi/llive
```

→ AI 関連を「余暇」と明示しつつ、本業との接続 (30 年経験) も示せる。リンクで深掘り可能。

---

## 案 C — 長い段落版 (~250 字、Eight に十分な自己紹介欄がある場合)

```
本業は精密計測・画像処理・三次元計測 (30 年超)。余暇では OSS の LLM フレームワーク
llive (FullSense umbrella の中核、Apache 2.0 + Commercial dual-license) を 1 人で
開発しています。Claude Code + 論文 RAG コーパス (49 分野・約 5 万件) + TRIZ + 30 年経験 を
組み合わせた「第二の脳」型の開発スタイルで、1 セッション 5 日間で 14 機能・1,276 テスト
全 PASS / 回帰ゼロを達成。長期的には精密計測の知見 (SH 係数の VQ 量子化など) を AI 側へ
還元したいと考えています。Repo: https://github.com/furuse-kazufumi/llive
```

→ 本業と趣味の接続点 (SH 係数 VQ など) を入れると、本業の精密計測クライアントが見ても違和感ない仕上がり。

---

## 推奨

**案 B** を第一候補に推します。

理由:
- 案 A は短すぎて、AI を「趣味で何かしている人」程度の印象で終わり、興味ある人が深掘りできない
- 案 C は名刺としては情報量が多すぎ、本業のクライアントには「何の人？」になりかねない
- 案 B は **「趣味で OSS 開発」** という前提を保ちつつ、**「本業との接続」** と **「興味ある人向けのリンク」** が揃う

「30 年の経験を AI 開発に活かす」表現は、本業の信用を損なわず、むしろ強化する方向に効くはず。

---

## 補足 — Eight 利用時の運用ヒント

- AI 関連の話題を **「趣味」「余暇」「副業」** ではなく **「研究テーマの 1 つ」** と書く手もあるが、Eight は名刺アプリなので本業を前面に出したほうが自然
- LinkedIn と Eight で言い回しを揃えると、両方を見られたときに整合性が出る (この記事の LinkedIn 投稿文も同じ「30 年経験 + 趣味 OSS」軸)
- 名刺交換後に LinkedIn / Qiita を見られた際、本記事の「第二の脳」シリーズが見えると印象が一段強まる (公開タイミングを名刺更新と合わせる選択肢あり)
