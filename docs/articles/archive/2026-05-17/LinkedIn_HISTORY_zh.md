# LinkedIn 投稿 — 2026-05-17 项目历史版 (简体中文版)

📜 **llmesh → llove → llive — FullSense 三个产品的开发历史和设计理念**

我 (古濑 和文 / puruyan) 研发的 **FullSense ™** umbrella 旗下三个 OSS 项目的足迹。

🧩 **llmesh** (最早) — secure LLM hub
- v1.5.0 完成 (MTEngine + XbarRChart + CUSUMChart + CLI)
- 设计: 通过 MCP 协议统一多个 LLM (OpenAI / Anthropic / Ollama 等)，嵌入 SPC，直连工业 IoT

🎮 **llove** (TUI HITL workbench)
- v0.2.2 PyPI 首发 (2026-05-09)
- F15 浏览器级显示 (Markdown / SVG / Mermaid)
- F16 多游戏 LLM 对局竞技场 (chess / go / mahjong / poker)
- F22 俄罗斯方块演示 / F23 PowerShell 兼容 shell / F24 Claude Code 集成
- F25 桥接层 (通过 MCP 连接 llove ↔ llmesh ↔ llive)
- 设计: 基于 Textual，准备多个演示「以动制胜」

🧠 **llive** (今天第 5 天) — 自进化型模块化记忆 LLM 框架
- 2026-05-13 启动, v0.1.0 (Phase 1 MVR)
- 2026-05-14: v0.3 → v0.4 → v0.5 (Phase 3/4/5 同日)
- 2026-05-16: v0.6.0 (9 axes skeleton + Apache 2.0 + FullSense umbrella)
- 今天: Brief API + 32 项需求 + 1,014 项测试通过

🎯 **6 大设计支柱**:
1. 本地优先 = AI 的本来归宿
2. 责任所在嵌入 architecture level
3. 多面向思考结构化 (10 思考因子)
4. 用 TRIZ 让创造成为结构
5. 诚实披露作为研究核心
6. 以家族方式开发 (3 个产品 1 个世界观)

⭐ **差异化要素**:
- vs 云 LLM (GPT/Claude/Gemini): 本地运行、审计 ledger、灾害模式胜出
- vs Agent 框架 (LangChain/CrewAI): TUI HITL、形式验证、TRIZ 胜出
- vs Wolfram Alpha: 完全 OSS、LLM 集成、审计日志胜出
